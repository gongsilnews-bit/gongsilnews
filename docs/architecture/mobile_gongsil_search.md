# 모바일 공실 검색 기능 기획서

> 작성일: 2026-05-02
> 상태: 기획 단계 (구현 전)

---

## 1. 목적

PC 공실 페이지(`GongsilClient.tsx`)에 구현된 **카테고리별 매물 검색 + 상세 필터** 기능을
모바일 환경(`/m/gongsil`)에 최적화된 UX로 구현한다.

---

## 2. 현재 상태

### PC 공실 (`GongsilClient.tsx`)
- **카테고리 탭**: 아파트·오피스텔 | 빌라·주택 | 원룸·투룸 | 상가·사무실·공장·토지 | 분양 | MY관심공실
- **서브 필 (Pills)**: 카테고리별 세부 유형 (아파트, 재건축, 오피스텔 등)
- **상세 필터**: 거래방식, 가격대, 면적, 사용승인일, 세대수, 층수, 방/욕실수, 방향, 기타옵션
- **2줄 구조**: 1줄 = 카테고리 탭, 2줄 = 서브 필/필터

### 모바일 공실 (`/m/gongsil`)
- 검색/필터 기능 **전혀 없음**
- 지도 위에 `🏢 공실 N건` 배지 + `내 위치` 버튼만 존재

---

## 3. 설계 원칙

### 원칙 1: 2줄 → 1줄 통합
PC에서는 카테고리 탭과 서브 필/필터가 2줄로 분리되어 있지만,
모바일에서는 **한 줄의 수평 스크롤 메뉴**로 통합한다.

### 원칙 2: 선택 시 하단 패널 확장 (네이버 스타일)
- 검색 메뉴(필 버튼)를 터치하면 **지도 아래쪽에 필터 옵션 패널**이 슬라이드업으로 나타남
- 네이버 부동산 앱과 동일한 UX 패턴 적용
- 패널 밖 영역 터치 또는 X 버튼으로 닫힘

### 원칙 3: 메뉴 수평 슬라이딩 (뉴스 탭바 스타일)
- 필터 메뉴가 화면 폭보다 많으므로 **좌우 스와이프 가능한 수평 스크롤바**로 배치
- 공실뉴스 카테고리 탭바(`MobileNewsClient.tsx`)와 동일한 UX
- 활성 메뉴는 강조 표시 (색상 변경 + 밑줄/배경)

### 원칙 4: 상단 헤더 텍스트 검색 (뉴스+공실 통합 검색)
- 헤더의 **🔍 아이콘**을 터치하면 텍스트 검색 입력 화면이 열림
- 현재는 **공실 매물 텍스트 검색** (건물명, 주소, 매물번호 등)
- 향후 **뉴스 기사 + 공실 매물 통합 검색**으로 확장 예정
- 모바일 뉴스의 검색 UX(`HomeHeader` 🔍 아이콘)와 동일한 진입점

## 4. UI 구조 설계

```
┌────────────────────────────────────────────┐
│ [공실등록]  부동산이 무료 열람하는...  🔍 ≡  │  ← 헤더 (🔍 = 텍스트 검색)
├────────────────────────────────────────────┤
│ [≡] ← [📍위치▼] [매물유형▼] [거래방식▼] ... → │  ← 통합필터 + 수평 스크롤 필터 바
├────────────────────────────────────────────┤
│                                            │
│              카카오 지도                     │
│                                            │
│  🏢 공실 N건              ◎ 내 위치         │
│                                            │
│  ┌──────────────────────────────┐          │
│  │ 매매 ✕  전세 ✕  빌라 ✕       │          │  ← 선택된 필터 태그 (하단 플로팅)
│  │ ↻ 초기화     [764개 매물 보기] │          │
│  └──────────────────────────────┘          │
└────────────────────────────────────────────┘
```

### 4-0. 통합 필터 버튼 (`≡` 아이콘)

필터 바 **맨 왼쪽**에 고정 배치되는 아이콘 버튼으로, 터치 시 **풀스크린 필터 패널**이 열린다.
개별 필 버튼이 한 가지 필터만 다루는 반면, 통합 필터는 **모든 필터를 한 화면에서 한꺼번에** 설정할 수 있다.

> 네이버 부동산 앱의 `≡` (필터 아이콘) 버튼과 동일한 패턴

#### 버튼 디자인
- **위치**: 필터 바 맨 왼쪽, 수평 스크롤 영역 **밖** (고정)
- **형태**: `≡` 아이콘 + 우측 세로 구분선 (`|`)
- **활성 상태**: 필터가 1개 이상 적용 중이면 아이콘에 빨간 점 뱃지 표시

#### 풀스크린 필터 패널 (통합 필터)

```
┌────────────────────────────────────────────┐
│            필터 ⓘ                    ✕     │  ← 상단 타이틀 + 닫기
├────────────────────────────────────────────┤
│  거래유형  매물유형  가격  면적  층수  방향 ... │  ← 앵커 탭 (수평 스크롤, 해당 섹션으로 점프)
├────────────────────────────────────────────┤
│                                            │
│  ■ 거래유형                                 │
│  ┌────┐ ┌────┐ ┌────┐ ┌──────┐            │
│  │매매│ │전세│ │월세│ │단기임대│            │
│  └────┘ └────┘ └────┘ └──────┘            │
│                                            │
│  ■ 매물유형                                 │
│  ┌────┐ ┌────┐ ┌──────┐                   │
│  │아파트│ │재건축│ │오피스텔│ ...             │
│  └────┘ └────┘ └──────┘                   │
│            더보기 ∨                         │
│                                            │
│  ■ 가격                              ∧     │
│  매매가 전체 ✏                              │
│  ○──────────────────────────○              │
│  최소    10억    20억    30억    최대        │
│                                            │
│  보증금 전체 ✏                              │
│  ○──────────────────────────○              │
│  ...                                       │
│                                            │
│  ■ 면적                                    │
│  ...                                       │
│                                            │
│  ■ 층수                                    │
│  ...                                       │
│                                            │
│  ■ 방/욕실                                  │
│  ...                                       │
│                                            │
│  ■ 방향                                    │
│  ...                                       │
│                                            │
│  ■ 옵션                                    │
│  ...                                       │
├────────────────────────────────────────────┤
│  매매 ✕  전세 ✕  빌라 ✕                     │  ← 선택된 필터 태그
│  ↻ 초기화            [764개 매물 보기]       │  ← 하단 고정 CTA
└────────────────────────────────────────────┘
```

#### 통합 필터 핵심 특징

| 항목 | 설명 |
|---|---|
| **앵커 탭** | 상단 수평 스크롤 탭 → 해당 섹션으로 스크롤 점프 (거래유형, 매물유형, 가격, 면적...) |
| **세로 스크롤** | 모든 필터 섹션이 하나의 스크롤 뷰에 연속 배치 |
| **선택 태그** | 하단에 현재 선택된 필터가 `태그 ✕` 형태로 표시, 개별 삭제 가능 |
| **초기화** | 모든 필터 한 번에 해제 |
| **적용 버튼** | `N개 매물 보기` — 실시간으로 매물 수 카운트 표시 |
| **닫기** | ✕ 버튼 또는 뒤로가기 → 필터 미적용 상태로 복원 (적용 전 취소) |

#### 통합 필터 vs 개별 필 버튼 관계

```
[≡ 통합필터]  = 모든 필터를 한 화면에서 설정 (풀스크린)
[매물유형▼]   = 매물유형만 빠르게 설정 (하단 패널)
[거래방식▼]   = 거래방식만 빠르게 설정 (하단 패널)
[가격▼]      = 가격만 빠르게 설정 (하단 패널)
...
```

- 통합 필터에서 설정한 값 ↔ 개별 필 버튼의 값은 **양방향 동기화**
- 어디서든 필터를 변경하면 나머지에도 반영됨

### 4-1. 수평 스크롤 필터 바

| 순서 | 메뉴 | 위치 | 터치 시 동작 |
|---|---|---|---|
| — | **≡ 통합필터** | 고정 (스크롤 밖) | 풀스크린 필터 패널 열기 |
| 0 | **📍 위치** | 스크롤 영역 | 지역 캐스케이딩 + 키워드 검색 (하단 패널) |
| 1 | **매물유형** | 스크롤 영역 | 매물유형 버튼 그리드 (하단 패널) |
| 2 | **거래방식** | 스크롤 영역 | 매매/전세/월세/단기임대 (하단 패널) |
| 3 | **가격** | 스크롤 영역 | 가격 범위 슬라이더 (하단 패널) |
| 4 | **면적** | 스크롤 영역 | 면적 범위 프리셋 (하단 패널) |
| 5 | **층수** | 스크롤 영역 | 층수 범위 선택 (하단 패널) |
| 6 | **방/욕실** | 스크롤 영역 | 방·욕실 개수 (하단 패널) |
| 7 | **방향** | 스크롤 영역 | 남/동/서/북향 (하단 패널) |
| 8 | **옵션** | 스크롤 영역 | 에어컨, 세탁기 등 체크박스 (하단 패널) |

> PC의 `CATEGORY_CONFIG`에 정의된 `basicFilters`를 참고하되, 
> 카테고리를 별도 탭으로 분리하지 않고 **매물유형 필터 안에 통합**한다.

### 4-2. 하단 필터 패널 (개별 필 터치 시)

- **위치**: 지도 하단에서 슬라이드업 (height: 최대 50vh)
- **배경**: 반투명 오버레이 (`rgba(0,0,0,0.3)`)
- **구조**: 제목 + 옵션 그리드 + [초기화 | 적용] 버튼
- **네이버 스타일**: 선택된 값은 필 버튼에 요약 표시 (예: `빌라 ▼` → `빌라, 원룸 ▼`)

### 4-3. 필터 적용 로직

```
사용자가 필터 선택 → [적용] 또는 [N개 매물 보기] 클릭
→ vacancies 배열을 조건별 .filter() 로 필터링
→ 필터링된 매물만 지도 마커로 표시
→ 클러스터 리스트에도 필터링 반영
→ 지도 하단에 선택 태그 플로팅 표시 (매매 ✕, 빌라 ✕ ...)
```

### 4-4. 위치 검색 (지역 네비게이션)

PC의 `MapSearchBar` 컴포넌트와 **동일한 기능**을 모바일 필터 바에 통합한다.

#### 현재 PC 동작 방식 (`MapSearchBar.tsx`)
```
┌──────────────────────────────────────────────┐
│  시/도 선택 ▼ │ - ▼ │ - ▼ │ 검색 🔍 ▼      │  ← 플로팅 바 (드래그 이동 가능)
└──────────────────────────────────────────────┘
         ↓ 클릭 시
┌──────────────────────────────────────────────┐
│  [시/도]  [시/군/구]  [읍/면/동]  ← 탭 전환   │
│  ┌────┐ ┌────┐ ┌────┐                        │
│  │서울│ │부산│ │대구│  ...                     │
│  └────┘ └────┘ └────┘                        │
│                                  [닫기]      │
└──────────────────────────────────────────────┘
```

- **시/도 → 구/군 → 동** 3단계 캐스케이딩 선택
- 행정구역 코드 API: `grpc-proxy-server-mkvo6j4wsq-du.a.run.app/v1/regcodes`
- 선택 시 → 카카오 Places API로 좌표 검색 → `map.panTo()` 지도 이동
- **키워드 검색**: 직접 텍스트 입력 → 카카오 장소 검색 → 결과 목록 → 선택 시 이동

#### 모바일 적용 방식

필터 바의 **첫 번째 필(📍 위치)**을 터치하면 하단 패널에 **2개의 탭**이 표시된다:

```
┌────────────────────────────────────────────┐
│  📍 위치 검색                         ✕    │
├────────────────────────────────────────────┤
│  [지역선택]          [키워드검색]            │  ← 탭 전환
├────────────────────────────────────────────┤
│                                            │
│  ■ 지역선택 탭                              │
│  ┌──────────────────────────────────────┐  │
│  │ [시/도 ▼]  [시/군/구 ▼]  [읍/면/동 ▼] │  │  ← 3단 드롭다운
│  └──────────────────────────────────────┘  │
│                                            │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐             │
│  │서울│ │경기│ │인천│ │부산│  ...          │  ← 3열 버튼 그리드
│  └────┘ └────┘ └────┘ └────┘             │
│                                            │
│  ■ 키워드검색 탭                            │
│  ┌──────────────────────────┐ ┌────┐      │
│  │ 동, 읍, 면 또는 랜드마크    │ │이동│      │  ← 검색 입력 + 결과 목록
│  └──────────────────────────┘ └────┘      │
│  📍 강남역                                  │
│  📍 삼성동                                  │
│                                            │
└────────────────────────────────────────────┘
```

#### 위치 검색 동작 흐름

```
1. 필터 바 "📍 위치" 터치
   → 하단 패널 열림 (기본: 지역선택 탭)

2-A. 지역선택 경로:
   시/도 선택 (예: 서울)
   → 지도가 서울 전역으로 이동 (줌 레벨 8)
   → 시/군/구 그리드 자동 로드
   → 구/군 선택 (예: 강남구)
   → 지도가 강남구로 이동 (줌 레벨 6)
   → 읍/면/동 그리드 자동 로드
   → 동 선택 (예: 역삼동)
   → 지도가 역삼동으로 이동 (줌 레벨 4)
   → 패널 자동 닫힘
   → 필 텍스트 업데이트: "📍 위치" → "📍 강남구 역삼동"

2-B. 키워드 검색 경로:
   검색어 입력 (예: "판교역")
   → Enter 또는 [이동] 터치
   → 카카오 Places API 검색
   → 결과 목록 표시
   → 결과 터치 → 지도 이동 (줌 레벨 5)
   → 패널 자동 닫힘
   → 필 텍스트 업데이트: "📍 위치" → "📍 판교역"
```

#### 재사용 가능 로직 (MapSearchBar.tsx에서 가져옴)

| 함수 | 역할 | 모바일 재사용 |
|---|---|---|
| `loadRegSido()` | 시/도 목록 API 호출 | ✅ 그대로 사용 |
| `loadRegGugun(sidoCode)` | 구/군 목록 API 호출 | ✅ 그대로 사용 |
| `loadRegDong(gugunCode)` | 동 목록 API 호출 | ✅ 그대로 사용 |
| `moveToMapSearchByKeyword(keyword, zlevel)` | 카카오 장소 검색 → 지도 이동 | ✅ 그대로 사용 |
| `executeMapKeywordSearch()` | 키워드 검색 → 결과 목록 | ✅ 그대로 사용 |

> **핵심**: `MapSearchBar.tsx`의 데이터 로직을 그대로 가져오되,
> UI만 모바일 하단 패널 형태로 재구성한다.

### 4-5. 상단 헤더 텍스트 검색 (🔍)

헤더 우측의 **🔍 아이콘**을 터치하면 열리는 **텍스트 기반 통합 검색** 기능.
현재 모바일 뉴스의 `HomeHeader` 🔍와 동일한 진입점을 사용한다.

#### 검색 화면 UI

```
┌────────────────────────────────────────────┐
│  ← [건물명, 주소, 매물번호 검색        ] 🔍  │  ← 검색 입력 바
├────────────────────────────────────────────┤
│                                            │
```

## 8. 컴포넌트 분리 및 리팩토링 설계 (Phase 2)

### 8-1. 배경 및 필요성
- 현재 `page.tsx`(약 1,170줄)와 `MobileFilterBar.tsx`(약 320줄)에 모든 UI와 필터 로직이 집중되어 파일 크기가 비대해짐.
- 향후 가격(최소/최대 입력, 15개 이상 프리셋 버튼), 면적, 사용승인일, 층수 등 복잡한 세부 필터가 추가되면 유지보수가 불가능해짐.
- 따라서 **기능별 패널(바텀 시트 내용물)을 독립된 컴포넌트로 분리**하는 리팩토링이 필수적임.

### 8-2. 디렉토리 및 컴포넌트 구조

```
src/app/m/gongsil/
 ├─ page.tsx                        (지도 렌더링 및 필터된 매물 마커 표시)
 ├─ MobileFilterBar.tsx             (수평 스크롤 컨테이너 및 껍데기 바텀 시트 관리)
 │
 └─ filters/                        (신규 폴더: 각 필터 패널 분리)
     ├─ LocationFilterPanel.tsx     (📍 지역 캐스케이딩 및 키워드 장소 검색)
     ├─ PropertyTypeFilterPanel.tsx (🏢 매물유형 그리드 다중 선택)
     ├─ TradeTypeFilterPanel.tsx    (🤝 거래방식 다중 선택)
     ├─ PriceFilterPanel.tsx        (💰 매매가/전세가/보증금 프리셋 및 직접 입력)
     ├─ AreaFilterPanel.tsx         (📐 면적 프리셋 및 평수 계산)
     ├─ DetailFilterPanel.tsx       (📅 층수, 사용승인일, 방/욕실 등 세부 조건)
     ├─ FullFilterOverlay.tsx       (≡ 통합 필터 풀스크린 팝업)
     └─ useVacancyFilters.ts        (필터 상태 및 필터링 로직 분리 Hook)
```

### 8-3. 역할 분담 (Responsibilities)

1. **`useVacancyFilters` Hook**
   - `FilterState` 인터페이스 정의 (가격, 면적, 층수, 연식 등 모든 세부 조건 포함).
   - 초기화 함수, 부분 업데이트 함수 등 상태 관리 로직 캡슐화.
   - `page.tsx`에 존재하던 `.filter()` 로직을 가져와 최종 `filteredVacancies` 반환.

2. **`MobileFilterBar.tsx`**
   - 현재 활성화된 패널 상태 (`activePanel`) 관리.
   - 하단 바텀 시트(`renderSheet`) UI 틀 제공.
   - 내부에 `activePanel` 값에 따라 `filters/*Panel.tsx` 컴포넌트들을 지연(lazy) 또는 조건부 렌더링.

3. **개별 Panel 컴포넌트**
   - `filters`(현재 상태)와 `onFilterChange`(상태 업데이트 함수)를 Props로 받아 내부 UI 렌더링.
   - 예: `PriceFilterPanel`은 1천, 3천, 1억 등 프리셋 버튼과 최소/최대 입력 폼만 렌더링하고, 값 변경 시 `onFilterChange` 호출.

### 8-4. 기대 효과
- **관심사 분리**: 각 파일이 100~200줄 내외로 유지되어 가독성과 유지보수성 향상.
- **재사용성**: 향후 PC 버전 개편이나 다른 메뉴에서 동일한 필터 UI를 재사용 가능.
- **성능 최적화**: 필터 패널 내용물을 별도 컴포넌트로 분리하면 불필요한 전체 리렌더링 방지 용이.
