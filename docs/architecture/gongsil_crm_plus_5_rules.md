# 📑 [공실 CRM 플러스] B2B2C 올인원 중개 솔루션 5대 대원칙 명세서
> 본 문서는 '공실뉴스' 플랫폼의 핵심 성장 동력인 B2B 고객관리(CRM)와 일반/B2B 문의하기 시스템의 완전 통합, 그리고 모바일 연동 및 원클릭 공실광고화를 실현하기 위한 **5대 골든 룰(Golden Rules)** 기획 및 설계 표준 사양서입니다.

---

## 🏆 Part 1. 공실 CRM 플러스 5대 골든 룰 (The 5 Golden Rules)

공실 CRM 플러스는 중개사가 매일 출근해서 퇴근할 때까지 PC와 스마트폰에서 단 1초도 끌 수 없는 공실뉴스의 핵심 엔진입니다.

```
┌────────────────────────────────────────────────────────────────────────┐
│                        공실 CRM 플러스 5대 골든 룰                        │
├────────────────────────────────────────────────────────────────────────┤
│  1. 모든 손님관리는 1곳에서! (임차인/임대인/부동산 통합 관리)              │
│  2. 문의 접수된 손님이 자동으로 저장! (Zero-Touch 리드 확보)             │
│  3. 손님에게 내가 만든 [AI 전단지 URL]을 선택해 문자/카톡 발송!          │
│  4. 접수된 물건을 원클릭으로 바로 공실광고 등록! (타이핑 제로 광속 등록)  │
│  5. 이 모든 완벽한 비즈니스 동선을 PC와 모바일에서 100% 동일 구현!       │
└────────────────────────────────────────────────────────────────────────┘
```

---

### 1️⃣ [Rule 1] 모든 손님관리는 1곳에서! (Single Source of Truth)
* **정의**: 임차인(손님), 임대인(건물주), 타 부동산(공동중개 파트너) 및 수동으로 직접 등록한 오프라인 워크인 고객까지 하나의 통합 CRM 대시보드에서 일원화 관리합니다.
* **중개사 가치**: 여러 창을 띄울 필요 없이, 오직 하나의 테이블에서 부동산의 모든 휴먼 네트워크 자산을 완벽 통제합니다.

### 2️⃣ [Rule 2] 문의 접수된 손님이 자동으로 저장! (Zero-Touch Lead Capture)
* **정의**: 매물 상세페이지, 모바일 스마트 전단지, 기자 미니홈피 등의 다이렉트 문의창을 통해 고객이 문의를 제출하는 즉시, 중개사가 직접 입력하지 않아도 자동으로 `crm_customers` 및 `crm_logs`에 **`[상태: 신규]`**로 매핑 저장됩니다.
* **중개사 가치**: 리드 수집 및 보존율 100% 달성, 수동 입력 리소스 완전 제거.

### 3️⃣ [Rule 3] 내가 만든 [AI 전단지 URL]을 선택해서 문자/카톡 발송! (Flyer-to-Client CRM Link)
* **정의**: 중개사가 **EasyFlyer AI** 엔진을 통해 제작해 둔 매물별 전단지 모바일 URL을 고객관리 상세창 내에서 원클릭으로 선택하여 손님에게 전송하는 기능입니다.
* **중개사 가치**: 비주얼 브리핑 퀄리티 극대화, 일반 텍스트 문자 대비 계약 성사율 10배 이상 상승.

### 4️⃣ [Rule 4] 접수된 물건은 원클릭으로 바로 공실광고 등록! (One-click Vacancy Advertising)
* **정의**: 임대인(건물주)이 임대 의뢰 문의를 남기면, 해당 의뢰 데이터를 그대로 매핑하여 **`[🏢 공실광고 등록]`** 양식의 주소, 보증금, 월세, 건물주 연락처 정보를 자동 완성해 즉시 광고로 온에어시킵니다.
* **중개사 가치**: 입력 오류 차단, 광고 발행 소요 시간 단 30초로 단축, 시장 매물 선점권 1위 확보.

### 5️⃣ [Rule 5] 이 모든 것을 PC와 모바일에서 100% 동일 구현! (Mobile Cross-device Parity)
* **정의**: 외근이 잦고 스마트폰 터치 응대가 잦은 중개사의 행동 패턴에 맞추어, 위의 모든 핵심 기능(CRM, AI 전단지 복사/발송, 원클릭 공실등록)을 모바일 브라우저/앱 화면에서도 완벽하게 동일 동작하도록 반응형 웹 디자인으로 밀착 설계합니다.
* **중개사 가치**: 공간 제약 없는 실시간 비즈니스 의사결정 가능.

## 🏃‍♂️ Part 2. 모바일 현장 리얼 업무 시나리오 (Mobile UX Flow)

외근 중인 김정민 공인중개사의 실전 하루를 통해 기획이 실제 가치로 실현되는 시나리오입니다.

```
[1. 실시간 모바일 리드 접수]
- 길에서 다른 매물 투어 안내 중에 진동 알림 🔔
- "임대인 박건물님이 서초동 102호 전세 3억 매물 의뢰를 하셨습니다."
       │
       ▼ (모바일 스마트폰 확인)
[2. 모바일 고객관리(CRM) 탭 열기]
- 공실뉴스 모바일 고객관리 리스트 최상단에 [상태: 신규] 빨간색 배지로 자동 등록 완료 확인.
       │
       ▼ (단 30초 원클릭 광고 발행)
[3. 현장에서 공실광고로 즉시 온에어]
- 상세보기 화면에서 [🏢 공실광고 즉시 등록] 버튼 터치.
- 임대인 성함/연락처/의뢰가격이 다 채워진 폼 확인 후, 현장에서 찍은 내부 사진 업로드 후 저장 클릭!
- 포털 메인에 실시간 공실광고 즉시 활성화 완료.
       │
       ▼ (B2C 브리핑 전송)
[4. 스마트 전단지 즉석 발송]
- 카톡으로 매물 문의를 준 또 다른 손님(임차인)에게 방금 올린 매물의 [AI 전단지]를 골라 [카톡 전송] 터치!
- 손님 스마트폰으로 프리미엄 모바일 제안서 URL 안전하게 도착.
```

## 💾 Part 3. 백엔드 데이터베이스 매핑 설계 (Database Mapping)

### 3.1. 통합 `crm_customers` 테이블 속성 정보
```sql
-- 5대 원칙을 완벽하게 수렴하기 위한 crm_customers 테이블 고도화
ALTER TABLE crm_customers ADD COLUMN IF NOT EXISTS client_role VARCHAR(50) DEFAULT 'tenant'; -- 'tenant' (임차), 'landlord' (임대), 'realtor' (공동중개)
ALTER TABLE crm_customers ADD COLUMN IF NOT EXISTS source_type VARCHAR(50); -- 'vacancy_detail', 'ai_flyer', 'reporter_homepage', 'general'
ALTER TABLE crm_customers ADD COLUMN IF NOT EXISTS meta_data JSONB DEFAULT '{}'::jsonb; -- B2B / 임대인 특화 JSON 메타데이터 저장소
```

### 3.2. 온라인 문의 ➡️ CRM 자동 저장 데이터 파이프라인
* **접수 채널**: 공실뉴스 포털, 스마트 전단지 (EasyFlyer AI)
* **저장 트리거**: `submitInquiry` API 액션 호출 시 `agencyId` 인자가 감지되면 ➡️ `crm_customers` 및 `crm_logs`에 자동으로 신규 고객 및 첫 메모 타임라인 로그를 인서트합니다.

## 📅 Part 4. 향후 시스템 고도화 로드맵
1. **[Phase 1] 3각 고객망 DB 확장 및 고객 모달 고도화**:
   * 임차인/임대인/부동산 구별 및 폰번호 자동 마스킹 및 해제 비즈니스 로직 연동.
2. **[Phase 2] AI 전단지 리스트 연계 및 제안 URL 발송 기능 개발**:
   * 고객관리 상세창 내 전단지 다중 선택 기능 및 프리미엄 제안서 모바일 웹 뷰어 구현.
3. **[Phase 3] 임대 의뢰 공실광고 즉시 등록 매핑 트리거 개발**:
   * 임대인 상세 정보창 내 `공실등록 연동` 버튼 매핑 파이프라인 구축.
4. **[Phase 4] 모바일 반응형 UX 고도화 및 소셜 알림톡 API 연동**:
   * 외근 시 원터치로 고객 관리 및 자동 감사의 알림톡 전송 시스템 통합.
