# 부동산회원 승인 절차 개선안

> **작성일:** 2026-05-17  
> **상태:** 회의 → 구현 예정  
> **목적:** 부동산회원 가입 승인 플로우를 PC/모바일 모두에서 명확하고 직관적으로 개선

## 1. 현재 시스템 진단 (AS-IS)

### 1.1 현재 승인 상태 체계
| 상태 | DB값 | 의미 |
|------|-------|------|
| 승인대기 | `PENDING` | 심사 요청됨, 관리자 검토 중 |
| 서류보완 | `REJECTED` | 관리자가 반려, 사유 제공 |
| 정상승인 | `APPROVED` | 최종 승인 완료 |

### 1.2 현재 플로우의 문제점

#### ❌ 문제 1: "저장"과 "심사 신청"이 구분되지 않음
- **PC (`MemberRegisterForm.tsx`):** `handleSubmit`에 `requestApproval` 파라미터가 존재하지만, 하단 버튼 UI에서 이를 명확하게 분리하여 노출하는 구조가 부족함
- **모바일 (`/m/admin/settings/page.tsx`):** 하단에 **`💾 저장하기`** 버튼 하나만 존재. 저장을 누르면 내부 로직에서 `REJECTED → PENDING` 전환 등이 암묵적으로 처리됨
- **사용자 혼란:** "내가 지금 심사를 요청한 건가? 아니면 임시 저장만 한 건가?" 구분 불가

#### ❌ 문제 2: 반려 사유가 모바일에서 명확하게 보이지 않음
- **PC:** `RealtorApprovalNotice.tsx` 컴포넌트로 대시보드 상단에 반려 사유를 노출하고 있음
- **모바일:** 기본정보 탭에 `statusLabel`(승인대기/서류보완/정상승인) 배지만 보이고, **구체적인 반려 사유(`reject_reason`) 텍스트가 노출되지 않음**
- 부동산정보 탭에도 반려 사유 안내가 없어 사용자가 무엇을 수정해야 하는지 모름

#### ❌ 문제 3: 심사 진행 상태의 흐름이 시각적으로 불분명
- 현재 어느 단계에 있는지(작성 중 / 심사 대기 / 반려 / 승인) 한눈에 파악하기 어려움
- PC와 모바일 간 상태 표현 방식이 다름

## 2. 개선 방향 (TO-BE)

### 2.1 핵심 원칙
1. **"임시 저장"과 "승인 신청"은 반드시 별도 버튼으로 분리**
2. **반려 사유는 PC/모바일 모두에서 동일하게 눈에 띄는 빨간 경고 박스로 표시**
3. **심사 상태의 시각적 단계 표시(Step Indicator)를 PC/모바일 공통으로 도입**
4. **모바일에서도 PC와 동등한 수준의 정보를 제공 (기능 누락 없음)**

### 2.2 상태별 사용자 경험 상세 설계

#### 📝 상태: 최초 작성 중 (신규 / 아직 심사 신청 안 한 상태)
```
┌─────────────────────────────────────────┐
│  📋 부동산회원 신청 정보를 입력해 주세요  │
│  모든 필수항목을 입력한 후               │
│  [승인 신청] 버튼을 눌러주세요            │
└─────────────────────────────────────────┘

하단 버튼:
┌──────────────────┐ ┌──────────────────────┐
│  💾 임시 저장     │ │  📋 승인 심사 신청하기 │
│  (회색/보조)      │ │  (파란색/메인 CTA)     │
└──────────────────┘ └──────────────────────┘
```

#### ⏳ 상태: 승인대기 (PENDING)
```
┌──────────────────────────────────────────────┐
│  ⏳ 현재 서류 검토가 진행 중입니다             │
│  관리자 확인 후 승인 처리됩니다 (보통 당일~1일)  │
└──────────────────────────────────────────────┘

하단 버튼:
┌──────────────────────────────────────────┐
│  💾 정보 수정 저장 (상태 유지)             │
│  (파란색, 단일 버튼)                      │
│  ※ 저장해도 승인대기 상태는 유지됩니다     │
└──────────────────────────────────────────┘
```

#### 🔴 상태: 서류보완 (REJECTED)
```
┌──────────────────────────────────────────────┐
│  🚨 심사 반려 - 서류 보완이 필요합니다        │
│                                              │
│  📌 반려 사유:                                │
│  "사업자등록증 사진이 흐릿하여 확인이           │
│   불가합니다. 다시 촬영하여 업로드해 주세요."    │
│                                              │
│  아래 정보를 수정한 후 [재심사 신청] 버튼을     │
│  눌러주세요.                                  │
└──────────────────────────────────────────────┘

하단 버튼:
┌──────────────────┐ ┌──────────────────────┐
│  💾 임시 저장     │ │  📋 수정 후 재심사 신청│
│  (회색/보조)      │ │  (주황색/메인 CTA)     │
└──────────────────┘ └──────────────────────┘
```

#### ✅ 상태: 정상승인 (APPROVED)
```
┌──────────────────────────────────────────────┐
│  ✅ 정상승인 완료                             │
│  부동산회원 서비스를 정상적으로 이용할 수       │
│  있습니다.                                    │
└──────────────────────────────────────────────┘

하단 버튼:
┌──────────────────────────────────────────┐
│  💾 정보 수정 저장                        │
│  (파란색, 단일 버튼)                      │
│  ※ 핵심정보 변경 시 재심사가 필요할 수     │
│    있습니다                               │
└──────────────────────────────────────────┘
```

## 4. PC ↔ 모바일 기능 동등성 매트릭스

| 기능 | PC (현재) | 모바일 (현재) | 개선 후 (양쪽 동일) |
|------|-----------|--------------|-------------------|
| 승인 상태 배지 | ✅ 있음 | ✅ 있음 | ✅ Step Indicator로 통일 |
| 반려 사유 표시 | ⚠️ 대시보드에만 | ❌ 미표시 | ✅ 부동산정보 탭 상단에 명확히 표시 |
| 임시 저장 버튼 | ❌ 없음 (저장=신청) | ❌ 없음 (저장=신청) | ✅ 별도 버튼으로 분리 |
| 승인 신청 버튼 | ⚠️ 존재하나 불명확 | ❌ 없음 | ✅ 명확한 CTA 버튼 |
| 재심사 신청 버튼 | ⚠️ 저장=재심사 | ⚠️ 저장=재심사 | ✅ 명확한 CTA 버튼 |
| 확인 다이얼로그 | ❌ 없음 | ❌ 없음 | ✅ "정말 신청하시겠습니까?" 확인 |
