# 부동산 고객관리(CRM) 시스템 아키텍처 및 DB 설계 구조

본 문서는 멀티 테넌트(SaaS) 기반 '공실뉴스' 부동산 플랫폼의 고객관리(CRM) 모듈 개발을 위한 데이터베이스 설계 원칙과 아키텍처를 정의합니다.

## 1. 아키텍처 설계 원칙 (SaaS 기반 다중 입주 플랫폼)

모든 부동산 중개소는 단일 데이터베이스를 공유하지만, 데이터는 철저히 격리되어야 합니다.
- **Tenant Isolation (격리)**: 모든 고객 데이터는 소속된 부동산의 고유 식별자(`agency_id`)를 가집니다.
- **Data Filtering**: 관리자가 로그인하여 고객 리스트를 호출할 때, 백엔드/API 단에서 항상 `where agency_id = ?` 조건을 강제하여 **자신의 고객만** 볼 수 있도록 보장해야 합니다.
- **Inbound Routing**: 웹사이트 홈페이지(예: `buildon.gongsilnews.com`)에서 새로 접수된 [문의남기기] 로그는 해당 서브도메인을 소유한 부동산의 `agency_id`를 매핑하여 DB에 저장됩니다.

---

## 2. 데이터베이스 모델링 (관계형 2-Table 구조)

고객 정보와 히스토리(메모)는 1:N 관계로 설계되며, JSON 칼럼 저장이 아닌 2개의 독립된 테이블로 분리하여 저장합니다. 이는 동시성 보장과 고속 필터링 연산을 지원하기 위함입니다.

### 2.1. 고객 기본 정보 테이블 (`crm_customers`)
고객의 불변/기본 속성을 저장하며, 빠른 검색(이름, 번호)과 필터링(상태, 유형)에 최적화된 형식입니다.

| 컬럼명 | 타입 | 설명 | 비고 |
| :--- | :--- | :--- | :--- |
| `id` | uuid / int | 고객 고유 패스워드키 | PK |
| `agency_id` | uuid | 해당 고객을 소유한 부동산 식별자 | FK (agencies) |
| `user_id` | uuid | 홈페이지에 가입한 유저인 경우 연동 | FK (auth.users), Nullable |
| `name` | string | 고객의 이름 | |
| `phone` | string | 연락처 (010-0000-0000) | |
| `type` | string | 매수, 매도, 임차, 임대 등 | |
| `status` | string | 신규, 진행중, 계약완료, 보류 등 | 현재 상태 기록용 |
| `budget` | string | 가용 예산 입력란 | |
| `area` | string | 희망 지역 또는 메모 | |
| `source` | string | 유입 경로 (워크인, 전화, 홈페이지 등) | |
| `created_at` | timestamp | 고객 최초 등록 일시 | 정렬 기준 |

### 2.2. 상담 로그 및 타임라인 테이블 (`crm_logs`)
상태 변경, 통화 기록, 문자 발송 내역, 수동 메모 등을 카카오톡 메시지처럼 날짜/시간별로 누적(Append-only) 저장합니다.

| 컬럼명 | 타입 | 설명 | 비고 |
| :--- | :--- | :--- | :--- |
| `id` | uuid / int | 로그 고유 키 | PK |
| `customer_id` | uuid / int | 이 메모가 속한 고객의 식별자 | FK (crm_customers) |
| `type` | string | 로그 유형 (`memo`, `system`, `sms_sent` 등) | UI 표시 스타일 분기용 |
| `content` | text | 상세 메모 내용 (ex: "논현동 매물 2건 투어") | |
| `created_at` | timestamp | 기록 작성 일시 | 정렬 기준 |

---

## 3. UI 및 UX 연동 설계 (프론트엔드)

1. **대시보드 (List View)**: `crm_customers`를 쿼리하여 상태 탭(신규/진행중 등) 별로 데이터를 분리하여 테이블로 표시합니다. 
2. **사이드 슬라이드 패널 (Detail View)**: 고객 행을 클릭하면 우측에서 `CustomerDetailPanel`이 열리면서 해당 `customer_id`를 가진 `crm_logs`를 시간 역순으로 조회하여 타임라인 형태로 표시합니다.
3. **신규 고객 등록 모달**: 오프라인에서 워크인으로 온 고객을 중개사가 직접 등록할 때 사용하는 빠른 입력 폼입니다.
