# 22. 부동산 특강 모바일 상세 및 결제 페이지 아키텍처 계획 (2026-05-02)

## 1. 개요
현재 PC 버전에 맞춰 고정 너비와 우측 플로팅 패널로 구현되어 있는 부동산 특강 상세 페이지(`study_read`)를 모바일 기기에서도 원활하게 이용하고 수강 결제까지 이어질 수 있도록(수강 시작하기 페이즈) 반응형 UI/UX 구조로 전면 개편합니다.
본 문서는 Class101, 인프런 등 최신 에듀테크 플랫폼의 검증된 모바일 UX 패턴을 적용하기 위한 세부 구현 계획을 담고 있습니다.

## 2. 주요 개편 방향

### 2.1 반응형 레이아웃 뼈대 구축 (PC 고정 해제)
- **기존:** `<main style={{ width: 1200 }}>` 와 `<div className="flex">` 구조로 PC 환경의 가로 배치에 종속됨.
- **변경:** `<main className="w-full max-w-[1200px] px-4 md:px-6">` 로 화면 너비에 맞게 유연하게 축소되도록 변경.
- **배치 전환:** `flex-col lg:flex-row` 속성을 부여하여, 모바일 환경에서는 콘텐츠가 위아래로 쌓이고(세로 배치), PC 환경(`lg` 이상)에서는 좌우로 나뉘는 반응형 구조 완성.

### 2.2 모바일 전용 상단 핵심 정보 영역 추가 (`lg:hidden`)
- 모바일 화면에서는 공간적 제약으로 우측 패널을 유지할 수 없습니다.
- **조치:** 영상/이미지 캐러셀 바로 아래에 모바일 환경에서만 렌더링되는 **[특강 제목, 별점, 할인율, 가격 정보]** 블록을 신설합니다.
- 이를 통해 모바일 사용자는 스크롤을 내리기 전에 특강의 핵심 정보를 즉시 인지할 수 있습니다.

### 2.3 우측 결제 패널의 모바일 숨김 처리 (`hidden lg:block`)
- 기존의 우측 Sticky 결제 패널 `<div className="shrink-0" style={{ width: 360 }}>` 은 모바일 화면에서 심각한 레이아웃 파괴(가로 스크롤 발생 등)를 유발합니다.
- **조치:** `className="hidden lg:block shrink-0"`을 적용하여 모바일 화면에서는 완전히 숨김 처리하고, 오직 PC 환경에서만 플로팅 결제 패널로 동작하도록 제한합니다.

### 2.4 모바일 전용 고정 하단 결제 바 (Fixed Bottom Bar) 추가
- 긴 특강 소개와 커리큘럼, 리뷰를 모두 읽고 난 뒤 결제를 위해 다시 위로 스크롤을 올리는 불편함을 없애야 합니다.
- **조치:** 모바일(`lg:hidden`) 환경에서 화면 최하단(`bottom-0`)에 항상 고정되어 떠 있는 **Sticky/Fixed 하단 바**를 구현합니다.
- **구성 요소:** 
  - 좌측: 수강료 요약 정보 또는 쿠폰 다운로드 아이콘
  - 우측: 눈에 띄는 **"특강 수강 시작하기" CTA 버튼**
- 스크롤을 방해하지 않도록 본문 영역 하단에 넉넉한 `padding-bottom`을 부여하여 안정적인 시야를 확보합니다.

## 3. 기대 효과
- **구매 전환율 극대화:** 모바일 화면 어디서든 항상 따라다니는 하단 결제 버튼(Fixed Bottom CTA)을 통해 구매 동선 최소화.
- **완벽한 크로스 브라우징:** PC의 유려한 플로팅 패널 디자인은 유지하면서, 모바일에서는 답답함 없는 직관적인 세로형 콘텐츠 탐색 환경 제공.
- **유지보수성 향상:** 미디어 쿼리(Tailwind 브레이크포인트)를 통한 명확한 UI 분리로 향후 기능 추가 시 충돌 방지.
