# 모바일 관리자(Admin) UI/UX 구현 아키텍처

본 문서는 공실뉴스 모바일 플랫폼 내에서 최고관리자 및 부동산 회원이 사용하는 관리자 페이지(기사관리, 고객관리 등)의 아키텍처 및 UI/UX 구현 방식을 기록합니다. 기존 PC 위주의 관리자 기능을 모바일에 최적화하여 이식하는 과정에서 확립된 패턴을 정리했습니다.

---

## 1. 개요 및 목적
PC 버전의 관리자 대시보드(`/admin`, `/realty_admin`)는 다수의 테이블과 모달, 복잡한 필터링 기능을 포함하고 있습니다. 이를 좁은 모바일 화면에 그대로 적용할 경우 사용성이 크게 저하되므로, **카드 기반의 모바일 친화적 인터페이스**로 전환하는 것이 핵심 목적입니다.

### 주요 이식 대상
- **기사관리 (`/m/admin/article`)**: 기자들이 작성한 기사의 조회 및 최고관리자 전용 승인/반려 기능.
- **고객관리 (`/m/admin/customer`)**: 부동산 회원의 상담 고객 CRM (신규등록, 상태변경, 메모 추가).
- **공실관리 (`/m/admin/vacancy`)**: 부동산 회원의 매물(공실) 등록 및 상태 관리.

---

## 2. 모바일 관리자 디자인 패턴 (Standard Pattern)

모바일 관리자 페이지는 아래와 같은 **단일 페이지 애플리케이션(SPA)** 디자인 패턴을 공통적으로 따릅니다. 서버 컴포넌트 간 이동을 최소화하고 클라이언트 측 상태(State) 전환을 활용해 부드럽고 빠른 체감을 제공합니다.

### 2.1. 뷰(View) 상태 전환
각 페이지는 내부적으로 `view` 상태를 가져 화면을 전환합니다:
```ts
type View = "list" | "create" | "detail";
const [view, setView] = useState<View>("list");
```
1. **List View (목록 뷰)**: 기본 화면. 필터링 가능한 카드 리스트 제공.
2. **Detail View (상세 뷰)**: 특정 카드 클릭 시 전체화면으로 전환되어 상세 정보 및 액션(메모 작성, 상태 변경 등) 제공.
3. **Create View (등록/작성 뷰)**: FAB(Floating Action Button)을 통한 신규 데이터 입력 폼 (전체화면).

### 2.2. 공통 UI 요소 구성
- **Sticky Header**: 뒤로가기(햄버거 메뉴 열기 `/m?menu=open`) 및 현재 페이지 타이틀, 진행 상태 건수 요약.
- **접이식 검색바 (Search Bar)**: 돋보기 아이콘을 누르면 나타나는 가로형 검색 필드.
- **가로 스크롤 탭 필터 (Scrollable Tabs)**: 
  - 상태별 필터(예: 전체, 신규, 진행중, 승인대기 등).
  - 터치 디바이스에 맞춘 가로 스와이프 인터페이스 및 각 탭별 건수(Badge) 표시.
- **정보 배너 (Info Banner)**: 페이지 상단에서 사용자의 권한 및 주요 가이드를 안내.
- **카드 컴포넌트 (Card List)**:
  - 데이터의 핵심 정보만 요약하여 표시 (상태 뱃지, 이름/제목, 연락처/날짜 등).
  - 즉각적인 액션 버튼 포함 (예: `📝 상세/메모`, `📞 전화`, `✅ 즉시승인`).
- **FAB (Floating Action Button)**: 화면 우측 하단 고정, 새로운 데이터를 등록하는 버튼.

## 3. 기능별 세부 구현 내역

### 3.1. 모바일 고객관리 (Mobile CRM)
- **위치**: `src/app/m/admin/customer/page.tsx`
- **사용자**: 부동산 회원 (`Realtor`), 최고관리자
- **주요 특징**:
  - 기존 PC 버전(`CustomerSection.tsx`, `CustomerModal.tsx`, `CustomerDetailPanel.tsx`)의 복잡한 3분할 뷰를 단일 컴포넌트 상태 뷰로 통합.
  - **전화 걸기 기능**: 모바일 환경의 특성을 살려 카드 및 상세 뷰에 `href="tel:{전화번호}"` 태그를 적용해 바로 통화 앱으로 연동.
  - **상담 타임라인**: 상세 뷰 하단에 상담 메모와 상태 변경 기록을 채팅 인터페이스처럼 시각화.
  - 기존 Server Action(`customer.ts`)을 100% 재사용하여 백엔드 수정 불필요.

### 3.2. 모바일 기사 승인/반려 시스템 (Mobile Article Approval)
- **위치**: `src/app/m/admin/article/page.tsx`
- **사용자**: 회원(기자 역할), 최고관리자(`ADMIN`)
- **주요 특징**:
  - 로그인된 회원의 `role`을 검사하여 **최고관리자**인 경우, 본인 기사뿐만 아니라 전체 회원의 기사를 조회.
  - 승인 대기 중인 기사에 대해 `✅ 승인`, `🚫 반려` 버튼 노출.
  - **반려 프로세스**: 모달 창을 띄워 자주 쓰이는 반려 사유 템플릿(4가지) 선택 또는 직접 입력 가능.
  - 승인 신청 기능은 일반 회원용으로 기존과 동일하게 유지.
  - 관련 DB 연동: `adminUpdateArticleStatus` (다중 ID 및 반려 사유 파라미터 처리 가능)

## 4. 네비게이션 및 메뉴 연동 아키텍처

모바일 내 관리자 네비게이션은 `GlobalDrawerMenu.tsx` (우측 햄버거 메뉴)를 통해 이루어집니다.

- **URL 기반 메뉴 상태**: 브라우저 히스토리(뒤로가기) 기능을 네이티브처럼 지원하기 위해 메뉴의 열림 상태를 로컬 state가 아닌 URL 쿼리스트링(`?menu=open`)으로 관리합니다.
- **모바일 관리자 라우팅**: 
  - 과거: PC 관리자 페이지(`/realty_admin?menu=customer`)로 연결
  - 개선: 모바일 환경 식별 후 모바일 전용 페이지(`/m/admin/customer`)로 직접 연결.

```tsx
// GlobalDrawerMenu.tsx 내 부동산회원 메뉴 구성 예시
const realtor = [
  { icon: '👥', label: '고객관리', desc: '상담 고객 목록', href: '/m/admin/customer' },
  { icon: '📝', label: '공실관리', desc: '나의 공실 현황', href: '/m/admin/vacancy' },
  ...
];
```

## 5. 결론 및 향후 확장 방안
이번 구현을 통해 수립된 "모바일 카드 + 단일 페이지 뷰 전환" 패턴은 모바일 관리자 페이지의 핵심 표준 템플릿이 되었습니다. 추후 **결제 관리**, **톡(Talk) 문의 관리**, **포인트 내역 관리** 등 다른 관리자 모듈을 모바일로 확장할 때도 이 템플릿의 UI 레이아웃과 상태 관리 구조를 그대로 복제하여 신속한 개발과 일관된 UX를 보장할 수 있습니다.
