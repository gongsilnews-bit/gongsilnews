# 공실등록 폼 (Vacancy Registration Form) UI/UX 스펙 및 로직 정리

본 문서는 플랫폼의 '공실등록' (PC/모바일) 화면에서 적용되는 카테고리별 맞춤형 UI 로직과 자동화 기능(건축물대장 연동, AI 마법사 등)의 최신 스펙을 정리한 명세서입니다. (최종 업데이트: 2026-06-02)

---

## 1. 카테고리별 동적 입력창 (옵션/테마) 자동 전환 로직

공인중개사들이 매물을 등록할 때, 불필요한 타이핑을 줄이고 가장 많이 쓰는 키워드들을 클릭만으로 선택할 수 있도록 대분류 및 소분류(Sub-category)에 따라 옵션/테마 버튼이 **동적으로 변경(Swap)**됩니다.

### 🏢 상업용 (상가, 사무실)
상가, 사무실, 건물/빌딩 등의 상업용 매물을 등록할 때 노출되는 전용 프리셋입니다.
* **추천 옵션**: 천장형에어컨, 내부화장실, 탕비실, 엘리베이터, 개별난방, 테라스, 주차가능
* **추천 해시태그(테마)**: #대로변, #가시성최상, #무권리, #카페추천, #병원추천, #인테리어완비, #역세권상가

### 🏭 산업/토지 (공장, 창고, 지식산업센터, 토지)
공장, 창고, 지식산업센터, 토지 등의 산업용 매물을 등록할 때 노출되는 전용 프리셋입니다.
* **추천 옵션**: 높은층고(5m이상), 호이스트, 마당넓음, 대형차량진입, 동력넉넉, 화물엘리베이터
* **추천 해시태그(테마)**: #IC인접, #도로접, #창고부지, #건축허가득, #즉시사용가능

> **개발 노트**: 이 로직은 `VacancyRegisterForm.tsx` (PC) 및 `page.tsx` (Mobile) 내에서 `useMemo`를 통해 `propertyType`과 `subCategory`의 상태값 변화를 감지하여 즉각적으로 프리셋을 다시 렌더링합니다.

---

## 2. 스마트 자동화 기능 및 예외 처리 로직

### 🏗️ 건축물대장 API 연동 자동완성
국토교통부 건축물대장 연동 API를 통해 주용도, 건물구조, 주차대수, 층수 등의 스펙을 원클릭으로 채워주는 기능입니다.
* **적용 카테고리**: 상가·사무실·건물·공장·토지 (단, '토지' 제외)
* **제외 카테고리**: 빌라·주택
  * **사유**: 주거용(빌라, 주택)의 경우 서류상 대장 정보와 실제 현황(위반건축물, 쪼개기 등)이 다른 경우가 많아, 플랫폼의 정보 신뢰도를 지키기 위해 자동완성(AI 연동) 및 부가 스펙 창을 폼에서 완전히 렌더링하지 않도록 제거했습니다. (중개사의 수동 입력 우선)

### ✨ AI 멘트 마법사 (자동 상세설명 생성)
Gemini AI를 활용하여 입력된 매물 스펙(가격, 면적, 옵션, 인프라 등)을 바탕으로 전문적인 마케팅 문구를 생성해 주는 기능입니다.
* **로딩 UX**: 생성 버튼 클릭 시 화면 전체에 블러 오버레이 팝업이 표시되며, 역동적인 회전 스피너와 그라데이션 프로그레스바가 애니메이션으로 동작하여 시각적 피드백을 제공합니다.
* **예외 처리**: AI 구동 중복 클릭을 막기 위해 실행 중에는 버튼이 비활성화(`disabled`) 됩니다.

## 3. 동적 UX 및 시각적 피드백(Loading) 로직

* **주변 인프라 탐색 로딩**:
  주소 등록 후 반경 1km ~ 5km 범위 내의 지하철, 편의점, 학교 등의 인프라를 카카오 API를 통해 재검색할 때, 검색 딜레이 타임 동안 시각적인 로딩 메시지(⏳)를 띄워 시스템이 정지된 것이 아님을 명확히 전달합니다.
  * **API 캐싱 무효화**: Kakao Local API 호출 시 동일한 쿼리로 인한 브라우저 캐시를 무시하고 최신 반경 데이터를 강제 조회하도록 `cache: "no-store"` 옵션을 적용했습니다.

* **지도 검색 필터(공실열람)**:
  투박한 텍스트 대신 **SVG 픽토그램(슬라이더 조절바)**과 `🔍 상세검색 ▼` 텍스트를 조합한 둥근 Pill 형태의 버튼 디자인을 채택하여, UI 전반의 디자인 통일성과 전문성을 극대화했습니다.

## 4. 모바일 플랫폼 동기화 (Responsive & Parity)
위에서 명시된 모든 데이터 정합성 규칙, 동적 테마 프리셋, 건축물대장 조건부 렌더링, 시각적 로딩 애니메이션 등은 PC 버전과 **모바일 어드민(`m/admin/vacancy/write/page.tsx`)** 간에 100% 동일한 로직으로 구현 및 동기화되어 있습니다.
