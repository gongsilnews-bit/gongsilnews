# 게시판 역할 기반 접근 제어(RBAC) 시스템

## 1. 개요
공실뉴스 플랫폼 내의 게시판(Board) 기능에 대해, 회원 `role`과 `plan_type`을 기반으로 한 0~5단계의 정교한 접근 권한(읽기, 쓰기, 답변 등) 제어 시스템을 구축하였습니다. 

이제 관리자는 각 게시판별로 "요구 레벨"을 자유롭게 설정할 수 있으며, 시스템은 접속한 유저의 레벨을 계산하여 접근 가능 여부를 실시간으로 판별합니다.

## 2. 권한 레벨 (0~5단계) 매핑 아키텍처
회원의 등급은 `src/utils/permissionCheck.ts`의 `getPermissionLevel` 함수를 통해 다음과 같이 매핑됩니다.

| 레벨 | 등급명 | 판별 조건 (DB `members` 테이블 기준) |
|---|---|---|
| **0** | 비회원 | 로그인하지 않은 상태 (`user === null`) 또는 `role` 값이 없는 경우 |
| **1** | 일반회원 | `role === 'USER'` 또는 인증된 회원이지만 특정 플랜이 없는 경우 |
| **2** | 무료부동산회원 | `role === 'REALTOR'`이며 유료 `plan_type`이 없는 경우 |
| **3** | 공실뉴스부동산 회원 | `role === 'REALTOR'`이며 `plan_type === 'news_premium'`인 경우 |
| **4** | 공실등록부동산 회원 | `role === 'REALTOR'`이며 `plan_type === 'vacancy_premium'`인 경우 |
| **5** | 최고관리자 | `role === 'ADMIN'` 또는 `role === '최고관리자'` |

## 3. 게시판별 권한 필드 관리 (관리자 모드)
관리자 메뉴의 "게시판 메뉴 (`/admin?menu=board`)" 에서 각 게시판별로 다음 3가지 핵심 권한을 독립적으로 부여할 수 있습니다.

- **목록보기권한 (`perm_list`)**: 게시글 목록 화면 진입 가능 여부
- **읽기권한 (`perm_read`)**: 특정 게시글 상세 내용을 볼 수 있는 권한
- **쓰기권한 (`perm_write`)**: 새로운 게시글 작성 권한 (수정/삭제 권한은 본인 및 관리자에게만 부여됨)

> 💡 **참고**: 기존에 혼란을 주던 "답변권한", "다운로드권한" 옵션은 UI/UX를 간소화하기 위해 2026. 04. 24. 업데이트를 통해 관리자 설정 양식(`BoardRegisterForm.tsx`)에서 일괄 제거되었습니다. 하지만 내부적으로 답변(`perm_reply`)은 기본값(1, 일반회원)으로 상속되어 처리됩니다.

## 4. 프론트엔드 (Client-Side) 권한 통제 가드(Guard)
각 화면(컴포넌트)별로 다음과 같은 가드 로직을 통해 사용자를 보호하고 비정상적인 우회 접근을 차단합니다.

### 4.1. BoardClient (게시물 목록 화면)
- **목록 진입 자체 차단 (`perm_list` 미달 시)**: 
  게시판 목록 접속 시도 시 화면 전체가 "접근 권한이 없습니다." 에러 페이지로 전환되고, \[메인으로 돌아가기\] 버튼이 활성화됩니다.
- **게시물 개별 클릭 시 블랙 토스트 알럿 (`perm_read` 검사)**:
  목록은 보이지만 "읽기 권한"이 없는 글을 누르면 밋밋한 브라우저 `alert()` 구조를 탈피하여, 즉각적으로 **블랙 토스트 메시지** ("OOO부동산 회원부터 열람하실 수 있습니다.. 🤍") 가 뜨며 본문 진입을 방어합니다.
- **쓰기 버튼 숨김 (`perm_write` 검사)**:
  해당 등급이 되지 않는 일반 열람자는 우측 상단의 \[글쓰기\] 버튼 자체가 화면에 그려지지 않습니다.

### 4.2. BoardReadClient (게시물 상세 화면)
- **URL 강제 진입 차단 (`perm_read` 미달 시)**: 
  목록을 거치지 않고 주소창에 `/board_read?id=...`를 직접 치고 들어오는 행위를 원천봉쇄합니다. 이 경우 "읽기 레벨: X레벨 이상 (현재 내 레벨: Y레벨)"을 명시하는 경고 화면이 표출된 뒤 튕겨냅니다.
- **답변 영역 통제 (`perm_reply` 미달 시)**: 
  기본 설정된 댓글 쓰기 권한이 없는 비회원이나 권한 미달 회원은 하단의 "댓글 입력창"과 버튼 UI 전체가 보이지 않아 원천적으로 댓글 작성을 차단합니다.

### 4.3. BoardWriteClient (새 글 쓰기 및 수정 화면)
- **쓰기 권한 검증 (`perm_write` 미달 시)**:
  글쓰기 에디터 컴포넌트 마운트 즉시 `canAccessBoard` 함수가 동작하여 권한 없음을 감지하고 에러 페이지를 렌더링, 관리자가 아닌 일반 유저의 악의적인 글 등록을 막습니다.

## 5. 설계 의의 (Design Philosophy)
1. **중앙화된 체크(Centralized Validation)**: `utils/permissionCheck.ts` 라는 싱글 스크립트에서만 0~5의 레벨 산정을 책임지므로 유지보수가 매우 간단해졌습니다.
2. **세련된 UX (Black Toast UI)**: 권한 부족 안내 상황에서 사용자에게 거부감을 줄 수 있는 투박한 팝업을 걷어내고, 부드러운 애니메이션(`toastFadeIn`)이 가미된 미니멀한 알림으로 사용자 감성을 더했습니다.
3. **견고한 백도어(Backdoor) 차단**: 프론트엔드의 화면 숨김뿐만 아니라 실제 데이터를 패치하는 `useEffect` 및 렌더링 사이클에서 강제 접근 시 차단막(Condition Rendering)을 세워 플랫폼 보안을 한 단계 끌어올렸습니다.
