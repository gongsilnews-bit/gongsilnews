# 공실검색 및 열람 데이터 Map

본 문서는 공실뉴스 플랫폼에서 **매물을 등록할 때 화면에 노출되는 동적 입력 조건(UI)**부터, 해당 입력값이 **지도 검색 화면에서 어떤 버튼으로 묶이고 노출되는지**까지 전체 플로우를 완벽하게 통합 정리한 명세서입니다. (2026-05-29 기준)

---

## 1. 카테고리별 매핑 및 노출 조건 (입력값 -> 지도 검색)
사용자가 공실을 등록할 때 선택하는 **'입력값(DB)'**이 지도 검색 화면에서 어떤 탭과 필터(버튼)로 매핑되는지 분석한 표입니다.

### 🏢 아파트·오피스텔
| 공실 등록 시 선택 (DB) | 지도 검색 필터 버튼 | 매핑 및 노출 상세 조건 |
| :--- | :--- | :--- |
| **아파트** | `[아파트]` | 1:1 개별 노출 |
| **오피스텔** | `[오피스텔]` | 1:1 개별 노출 |
| **아파트분양권** | `[기타]` | 비주력 4개 항목을 '기타' 버튼 하나로 묶어서 병합 노출 |
| **오피스텔분양권** | `[기타]` | 〃 |
| **재건축** | `[기타]` | 〃 |
| **재개발** | `[기타]` | 〃 |

### 🏘️ 빌라·주택
| 공실 등록 시 선택 (DB) | 지도 검색 필터 버튼 | 매핑 및 노출 상세 조건 |
| :--- | :--- | :--- |
| **빌라/연립** | `[빌라/연립]` | 1:1 개별 노출 |
| **단독/다가구** | `[단독/다가구]` | 단독/다가구 카테고리 노출 |
| **상가주택** | `[단독/다가구]` | '상가주택' 버튼을 삭제하고 '단독/다가구' 버튼에 병합 노출 |
| **전원주택** | `[전원주택]` | 1:1 개별 노출 |

### 🛏️ 원룸·투룸(풀옵션)
| 공실 등록 시 선택 (DB) | 지도 검색 필터 버튼 | 매핑 및 노출 상세 조건 |
| :--- | :--- | :--- |
| **원룸** | `[원룸]` | 1:1 개별 노출 |
| **투룸** | `[투룸]` | 1:1 개별 노출 |
| *(카테고리 무관)* | `[오피스텔만 보기]` | 공실 등록 시 **'테마'에 '오피스텔'을 체크한 매물만** 단독 노출 (상호 배타적 버튼) |

### 🏬 상가·사무실·건물·공장·토지
| 공실 등록 시 선택 (DB) | 지도 검색 필터 버튼 | 매핑 및 노출 상세 조건 |
| :--- | :--- | :--- |
| **상가** | `[상가]` | 1:1 개별 노출 |
| **사무실** | `[사무실]` | 사무실 카테고리 노출 |
| **지식산업센터** | `[사무실]` | '지식산업센터' 버튼을 삭제하고 '사무실' 버튼에 병합 노출 |
| **건물/빌딩** | `[건물/빌딩]` | 과거 데이터인 '건물', '빌딩/건물' 값까지 모두 포함하여 통합 노출 |
| **공장/창고** | `[공장/창고]` | 1:1 개별 노출 |
| **토지** | `[토지]` | 1:1 개별 노출 |

### 🏗️ 신축/분양 & 경매/공매
| 대분류 | 공실 등록 (입력/연동) | 지도 검색 필터 버튼 | 매핑 및 노출 상세 조건 |
| :--- | :--- | :--- | :--- |
| **신축/분양** | 아파트, 오피스텔 등 6개 | `[입력값과 동일]` | 하위 6개 항목 모두 버튼과 1:1 완벽 대응 |
| **경매/공매** | 외부 API 연동 (직접입력X) | `[아파트]`, `[빌딩]` 등 | 외부 데이터의 용도 키워드를 파싱하여 해당하는 지도 버튼에 조건부 노출 |

## 2. 1차 카테고리에 따른 동적 입력 폼(UI) 노출 조건
대분류 선택에 따라 '주거형' 매물인지 '상업용' 매물인지 시스템이 판단하여 하단 입력 폼을 다르게 보여줍니다.

### A. 주거형 매물 입력 폼
*   **적용 카테고리**: `아파트·오피스텔`, `빌라·주택`, `원룸·투룸`, `분양`
*   **노출되는 항목**: 
    *   `방 개수` (드롭다운 선택)
    *   `욕실 개수` (드롭다운 선택)
    *   `방향` (남향, 동향 등 거실 주실 기준 선택)
*   **숨김 처리되는 항목**: 해당층, 전체층

### B. 상업용 매물 입력 폼 (`COMMERCIAL_CATEGORY`)
*   **적용 카테고리**: `상가·사무실·건물·공장·토지`
*   **노출되는 항목**: 
    *   `해당층` (텍스트 입력: 예 - 2, 지하, 고층)
    *   `전체층` (텍스트 입력: 예 - 5)
*   **숨김 처리되는 항목**: 방 개수, 욕실 개수, 방향

## 3. 거래 유형에 따른 금액 입력 필드 로직
사용자가 선택하는 거래 방식(매매, 전세, 월세, 단기)에 따라 가격 입력칸의 레이블과 개수가 실시간으로 변경됩니다.

| 거래 유형 | 노출되는 금액 입력창 | 비고 (지도 검색 연동) |
| :--- | :--- | :--- |
| **매매** | `매매가` (1개) | 지도 검색의 '거래유형' 필터 중 `[매매]` 선택 시 검색됨 |
| **전세** | `전세 보증금` (1개) | 지도 검색의 `[전세]` 선택 시 검색 |
| **월세** | `보증금` + `월세` (2개) | 지도 검색의 `[월세]` 선택 시 검색 |
| **단기** | `보증금` + `월세` (2개) | 지도 검색의 `[단기임대]` 선택 시 검색 |

*(공통 사항: 관리비 입력 필드는 거래 유형과 무관하게 항상 노출됨)*

---

## 4. 주소 노출 범위 설정 및 면적 계산
*   **주소 노출 범위**:
    *   `동/호수 공개` (전체 공개)
    *   `동수 공개` (일부 공개)
    *   `비공개` (기본 주소, 즉 도로명/지번까지만 지도에 노출됨)
*   **면적 입력기**:
    *   `공급면적`, `전용면적` 입력 가능.
    *   평(py) 단위와 제곱미터(㎡) 단위가 자동 환산(면적 계산기)되어 양방향으로 동기화 입력됨.

---

## 5. 옵션 및 테마 (해시태그) 연동 조건
*   **옵션 (가전/가구 등)**: 
    *   에어컨, 세탁기, 냉장고 등 기본 체크리스트 제공.
    *   기타 옵션은 직접 입력하여 칩(Chip) 형태로 추가 가능.
*   **테마 (해시태그)**:
    *   #신축급, #올수리, #풀옵션 등의 태그.
    *   **★ 중요 연동**: 여기서 `#오피스텔`을 직접 입력하거나 선택할 경우, 1차 분류가 '원룸·투룸'일 때 지도 검색의 **`[오피스텔만 보기]`** 버튼 활성화 시 해당 매물이 화면에 나타나게 됩니다.

---

## 6. 기타 부가 기능
*   **AI 텍스트 자동 생성**: 위에서 입력한 `거래유형`, `금액`, `층수`, `면적`, `옵션`, `테마` 등의 모든 값을 프롬프트 페이로드로 묶어 서버로 전송, 중개사 전달사항 란에 '매물 설명 글'을 자동으로 생성해 주는 기능 포함.
