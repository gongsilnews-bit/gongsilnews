# 21. 부동산 특강 리뷰 UI 아키텍처 및 고도화 문서 (2026-05-02)

## 1. 개요
부동산 특강 상세 페이지(`study_read`) 내의 **수강생 리뷰 영역**의 사용성과 디자인을 대폭 개선한 내역을 기록합니다.
주요 목표는 소셜 로그인 사용자의 프로필 동기화, 모바일/PC 반응형 레이아웃 개선, 그리고 동적인 애니메이션(롤링 슬라이드, 아코디언 드롭다운)을 통한 사용자 경험(UX) 극대화입니다.

## 2. 주요 개선 사항

### 2.1 리뷰 작성 폼 프로필 이름 동기화
- **문제점:** 구글, 카카오 등 소셜 간편 로그인으로 가입한 사용자의 경우 `auth` 메타데이터(예: '공실뉴스')가 노출되어, 부동산 회원가입 시 기입한 실제 본명(`members.name`)과 불일치하는 문제 발생.
- **개선안:** 
  - `study_read/page.tsx` 내 `useEffect`에서 `supabase.auth.getUser()` 호출 시, 반환된 `user.id`를 기반으로 `members` 테이블을 즉시 조회.
  - 리뷰 작성 폼의 불필요한 프로필 이미지 아이콘을 제거하고, `members` 테이블에서 조회된 **실제 회원 이름** 텍스트만 깔끔하게 노출하도록 UI 단순화.

### 2.2 리뷰 리스트 자동 롤링 슬라이드
- **문제점:** 기존의 가로 스크롤(Horizontal Scroll) 방식은 우측으로 길게 늘어져 있어(파묻힘 현상), 리뷰를 한눈에 파악하기 어렵고 가독성이 떨어짐.
- **개선안:**
  - **Grid 레이아웃 적용:** 한 줄에 3개씩(PC 기준) 노출되도록 `grid-cols-1 sm:grid-cols-3`을 적용하여 직관적인 바둑판 형태로 변경.
  - **자동 롤링(Carousel) 구현:** `reviewStartIndex` 상태값과 `setInterval`을 활용하여 **2초마다 3개의 리뷰가 부드러운 페이드인(Fade-in) 애니메이션과 함께 스와이프** 되도록 동적 UI 구현.

### 2.3 인라인 아코디언 리뷰 전체보기
- **문제점:** "리뷰 전체 보기" 클릭 시 별도의 모달(팝업) 창이 화면을 가려 맥락이 단절되고 답답한 느낌 제공.
- **개선안:**
  - `isExpandedReviews` 상태를 도입하여, 버튼 클릭 시 기존 슬라이드 영역이 아래로 자연스럽게 늘어나며(Inline Expansion) 전체 리뷰 목록이 수직으로 표출되도록 아코디언(드롭다운) 방식으로 교체.
  - 최하단에 **"리뷰 숨기기"** 버튼을 배치하여 언제든 간편하게 닫고 기존의 자동 롤링 슬라이드 상태로 원복 가능.

### 2.4 Sticky 네비게이션 여백(Top Offset) 미세 조정
- **문제점:** PC 환경에서 화면 스크롤 시, 상단의 글로벌 블랙 헤더 라인에 의해 우측의 결제 패널이나 좌측의 탭 내비게이션(클래스 소개/커리큘럼 등)의 상단이 잘리는 현상 발생.
- **개선안:**
  - **우측 플로팅 결제 패널 (Class101 스타일):** 여유로운 공간을 선호하는 사용자 피드백에 따라 `top` 속성을 `136px`에서 **`180px`**로 확 늘려, 스크롤을 내려도 상단 헤더에 부딪히지 않고 안전한 여백을 확보.
  - **좌측 탭 네비게이션:** 탭의 경우 스크롤 시 뒷배경이 보이면 안 되므로, 헤더 높이에 딱 맞게 **`136px`**로 유지시켜 빈틈없는 밀착감 유지.

## 3. 기대 효과
- **데이터 정합성 보장:** 소셜 로그인 메타데이터 오류와 무관하게 모든 서비스 영역에서 동일한 실명 노출.
- **화면 몰입도 증가:** 자동 슬라이드와 자연스럽게 열리는 아코디언 확장을 통해 현대적인(Modern) UX 제공.
- **레이아웃 안정성:** 컴포넌트별 특성을 고려한 디테일한 Sticky Offset 분리 관리를 통해 스크롤 레이아웃 깨짐 현상 원천 차단.
