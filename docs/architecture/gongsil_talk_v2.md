# GongsilTalk V2 (공실Talk V2) 최종 기획 및 아키텍처 문서

## 1. 개요 (Overview)
기존 '매물/기사 댓글' 시스템과 완전히 분리된 독자적인 **부동산 중개사 전용 실시간 하이브리드 메신저(1:1 DM + 다중 오픈채팅)**를 구축합니다.
카카오톡, 텔레그램과 같은 직관적인 사용성을 제공하되, 디자인은 공실뉴스의 메인 컬러(블루/네이비)를 차용하여 프로페셔널한 B2B 중개망의 경험을 선사합니다.

## 2. 주요 기능 및 기획 (Key Features)

### 2.1 사이트 연동 및 레이아웃 (UI/UX)
*   **플로팅 버튼 진입:** 사이트 전역 우측 하단에 고정된 `[💬 공실Talk]` 둥근 버튼을 통해 언제든 접근합니다.
*   **카카오톡 스타일 오버레이:** 버튼 클릭 시 우측에서 카카오톡 PC 버전 크기(약 360x600px)의 패널이 슬라이드 오픈됩니다.
*   **화면 구성:** 좌측 슬림 LNB(내비게이션: 친구/채팅방/알림), 중앙 리스트 영역, 우측 대화방 영역으로 구성.
*   **브랜딩:** 익숙한 구조를 취하되 노란색 대신 공실뉴스 브랜드 컬러(네이비/블루)를 사용하여 전문성을 부여합니다.

### 2.2 메신저 핵심 기능 전개 전략
기존 플랫폼과 시너지를 낼 수 있도록 권한과 BM(수익모델)을 영리하게 분리합니다.

1.  **자유로운 1:1 다이렉트 메시지 (무료 개방)**
    *   모든 회원(일반 회원 포함)에게 무료로 개방하여 외부 메신저 이탈을 막고 플랫폼 락인(Lock-in) 효과를 극대화합니다.
2.  **다중 단체 채팅 (텔레그램/오픈채팅 스타일)**
    *   누구나 검색하여 입장할 수 있는 오픈 채팅방(예: "강남 상가 교류방")을 지원합니다.
    *   **권한 체계 (Role Management):** 
        *   👑 **방장 (Owner):** 방 개설자. 방 폭파, 부방장 임명, 전체 공지, 불량회원 강퇴 권한.
        *   🛡️ **부방장 (Admin):** 방장이 위임. 공지, 강퇴 등의 방 관리 권한.
        *   👤 **일반 참여자 (Member):** 메시지 송수신.
3.  **수익화 (BM) 연계**
    *   **방 개설 권한 제한:** 단체 채팅방 개설(방장 권한)은 `공실뉴스부동산` 또는 `공실등록부동산(유료)` 등 프리미엄 회원에게만 부여하여, 유료 결제 전환의 핵심 메리트로 동작하게 만듭니다.

### 2.3 매물 연동 및 프로필 (비즈니스 특화)
*   **매물 공유 기능:** 공실뉴스 매물을 보다가 클릭 한 번에 사진+가격+링크가 포함된 예쁜 카드 형태로 단톡방/1:1방에 전송(공유)할 수 있습니다.
*   **프로필(명함) 노출:** 상단이나 프사 클릭 시, 상대방의 '공실뉴스 프로필 명함(상호명, 연락처, 보유 매물 링크)'이 즉각적으로 팝업 노출되어 중개사 간 신뢰도를 높이고 원활한 거래를 돕습니다.

## 3. 데이터베이스 스키마 설계 (Safe Migration)

기존 댓글 방식(article_comments 등)에 덮어씌우지 않고 100% 무공해 신규 테이블을 설계하여 기 구축 시스템 결함(꼬임)을 원천 차단합니다. 
(※ 기존 [공실Talk]이라는 명칭은 UI 상에서만 [댓글/문의]로 수정합니다.)

1.  `talk_rooms` (채팅방 테이블)
    *   `id` (UUID) 
    *   `type` (VARCHAR): `private`(1:1), `group`(오픈채팅)
    *   `title` (VARCHAR): 방 이름 (그룹방 한정)
    *   `created_by` (UUID): 방 개설자 (가입 등급 체크용)
    *   `created_at` (TIMESTAMP)
2.  `talk_participants` (채팅방 참여자 기록 테이블)
    *   `room_id` (UUID)
    *   `member_id` (UUID)
    *   `role` (VARCHAR): `owner`, `admin`, `member` (권한 제어용)
    *   `last_read_at` (TIMESTAMP): 안읽음 뱃지 처리를 위한 마지막 읽은 시간
    *   `joined_at` (TIMESTAMP)
3.  `talk_messages` (채팅 메시지 원본 테이블)
    *   `id` (UUID)
    *   `room_id` (UUID)
    *   `author_id` (UUID)
    *   `content` (TEXT)
    *   `parent_id` (UUID): 특정 메시지에 대한 답장/인용 시 부모 ID
    *   `created_at` (TIMESTAMP)

*   **핵심 통신 인프라:** Supabase Realtime 채널 구독(Subscribe)을 활용, `talk_messages` 테이블의 INSERT 이벤트를 감지하여 지연 없는 실시간 채팅을 구현합니다.
