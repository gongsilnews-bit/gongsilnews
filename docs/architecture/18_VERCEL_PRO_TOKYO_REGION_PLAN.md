## 🚨 현재 문제: Vercel Hobby 플랜의 속도 병목

로컬에서는 빠르지만 Vercel 배포 환경에서 느린 원인은 **물리적 네트워크 거리**입니다.

```
현재: 브라우저(한국) → Vercel 서버(미국 워싱턴DC) → Supabase DB(한국)
                        ↑ 태평양 왕복 ~200ms              ↑ 태평양 왕복 ~200ms
                        
총 네트워크 지연: ~400ms (데이터 한 번 가져올 때마다)
```

### 병목 3요소
1. **물리적 네트워크 Latency**: Vercel(미국) ↔ Supabase(한국) 간 태평양 왕복 ~200ms
2. **Serverless 콜드 스타트**: Hobby 플랜은 서버가 자주 잠들어 부팅 지연 ~1초
3. **서버리스 타임아웃 10초**: 대용량 업로드 시 실패 위험

---

## 💡 해결책: Vercel Pro + 도쿄 리전

```
변경: 브라우저(한국) → Vercel 서버(도쿄) → Supabase DB(한국)
                        ↑ 한일 ~30ms           ↑ 한일 ~30ms
                        
총 네트워크 지연: ~60ms (약 7배 빨라짐!)
```

---

## 📊 속도 비교표

| 구간 | 로컬 | Vercel Pro(도쿄) | Vercel Hobby(미국) |
|------|------|-----------------|-------------------|
| 브라우저 → 서버 | **0ms** (같은 PC) | **~30ms** | ~200ms |
| 서버 → Supabase DB | **~30ms** (한→한) | **~30ms** (일→한) | ~200ms |
| 서버 콜드스타트 | **없음** | 가끔 ~100ms | 자주 ~1,000ms |
| **총 체감** | **⚡ 즉시** | **✅ 거의 즉시 (~100ms)** | **❌ 느림 (~500ms~1s+)** |

## 🛠️ 적용 방법

### 1단계: Vercel Pro 결제
- Vercel 대시보드 → Settings → Billing → Pro Plan 업그레이드

### 2단계: 리전 설정 (vercel.json)
```json
{
  "regions": ["hnd1"]
}
```
- `hnd1` = 도쿄 (한국에서 가장 가까운 Vercel 리전)

### 3단계: 재배포
- 설정 변경 후 자동 재배포 또는 수동 재배포

## 🎯 결론

- **월 $20으로 체감 속도 7배 향상** (가성비 최고)
- 로컬과 거의 동일한 속도 기대 (30ms vs 0ms는 체감 불가)
- 코드 최적화만으로는 물리적 거리(태평양 왕복)를 줄일 수 없음
- **Vercel Pro + 도쿄 리전이 가장 즉각적이고 극적인 효과**를 볼 수 있는 조치
