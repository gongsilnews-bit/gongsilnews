# Banner Management System Implementation Plan

## 1. 개요 (Overview)
관리자가 외부 도움 없이 직접 웹사이트(PC/모바일) 곳곳의 배너와 팝업 광고를 등록하고, 게재 기간과 노출 위치를 컨트롤할 수 있는 **[광고/배너 관리] 시스템**을 구축합니다. 화면에서 보내주신 NDSOFT CMS의 배너 관리 기능의 핵심을 Next.js 환경에 맞게 고도화하여 제안합니다.

## 2. 주요 기능 및 제안 (Key Features & Proposal)

### A. 배너 슬롯(Slot) 기반 위치 지정
*   **제안 내용:** 프론트엔드 화면(예: 메인 상단, 뉴스 기사 우측 사이드바, 공실 목록 하단 등)에 заранее **배너 슬롯(자리)** 을 지정해 둡니다.
*   관리자 페이지에서는 배너를 등록할 때 "어느 화면, 어느 위치(슬롯)에 띄울 것인지"와 "PC/모바일 중 어디에 띄울 것인지" 선택만 하면, 코딩(또는 에디터 수정) 없이 해당 위치에 노출됩니다.

### B. 광고(배너) 스케줄링 및 상태 관리
*   **스케줄링:** 시작 일시와 종료 일시를 설정하여, 이벤트 기간에만 자동으로 배너가 뜨고 내려가도록 자동화합니다.
*   **수동 제어:** '진행 상태 (진행/종료)' 버튼을 두어 기간과 무관하게 언제든 강제로 내리거나 올릴 수 있습니다.

### C. 상세 설정 (링크 및 디자인 속성)
*   이미지 업로드 및 썸네일 미리보기 지원
*   클릭 시 이동할 링크 URL 제공, 이동 방식(새창 열기 `_blank`, 현재창 이동 `_self`) 지정
*   상단/하단 여백(Margin)을 관리자가 직접 px 단위로 조절하여 화면의 여백을 미세 조정 가능

### D. 팝업(레이어/윈도우) 특별 지원
*   일반적인 배너 외에도 메인 접속 시 띄워지는 '팝업' 형태도 같은 메뉴에서 관리할 수 있도록 `팝업 전용 위치(Slot)`를 제공합니다. 하루 동안 보지 않기 쿠키 기능 등을 함께 연동합니다.

## 3. 기술 스택 및 데이터베이스 구조 제안 (Architecture)

### DB 테이블 (`banners`) Schema
- **`id`**: 배너 고유 ID (UUID)
- **`title`**: 배너명 (관리자 식별용)
- **`image_url`**: 배너 이미지 스토리지 경로
- **`link_url`**: 클릭 시 이동할 타겟 링크
- **`link_target`**: `_blank` / `_self` 
- **`placement_code`**: 노출될 슬롯 위치 (예: `MAIN_TOP`, `SIDEBAR`, `LIST_INLINE`, `POPUP`)
- **`device_type`**: `PC` / `MOBILE` / `ALL` (기기 타겟팅)
- **`start_time`**, **`end_time`**: 노출 시작/종료 일시
- **`is_active`**: Boolean (수동 강제 제어)
- **`margin_top`**, **`margin_bottom`**: 상하 여백값
- **`sort_order`**: 동일 위치에 배너가 2개 이상일 경우 정렬 순서

### 프론트엔드 연동 방식
```tsx
// 프론트엔드 개발 시 특정 영역에 슬롯 컴포넌트만 심어둡니다.
<BannerSlot placement="MAIN_TOP" />
```
*   `BannerSlot` 내부에서 현재 유효시간 (`start_time` <= 현재 <= `end_time`) 안에 속하고, `is_active`가 `true`인 배너들을 가져와 출력합니다.

## 4. 성능 최적화 전략 (Performance & Speed)
이 시스템을 도입하더라도 기존 사이트 로딩 속도에 지장을 주지 않도록 아래와 같이 최적화합니다.
*   **서버 캐싱 (ISR):** 배너 데이터를 매번 DB에서 조회하지 않고 Next.js의 캐싱 기능을 이용해 1~5분 단위로 갱신하여 DB 부하를 차단합니다.
*   **이미지 최적화 기능:** 업로드된 배너 이미지는 Next.js의 `<Image />` 컴포넌트를 이용해 WebP 포맷으로 압축되고, 스크롤을 내려 배너가 보이는 시점에만 이미지가 로드되도록(Lazy loading) 처리하여 초기 화면 렌더링 속도를 보장합니다.

4.  **[API/서버 액션]**: 배너 C-R-U-D 처리 로직 구축
5.  **[사용자 화면 연동]**: 사용자용 레이아웃에 `<BannerSlot />` 컴포넌트 개발 및 붙이기
