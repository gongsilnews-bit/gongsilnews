# 모바일 환경 UX/UI 최적화 내역 (2026-05-05)

본 문서는 공실뉴스 모바일 웹앱 환경에서 사용자 경험(UX)을 극대화하기 위해 적용된 스크롤 유지 및 모달 내비게이션 개선 사항을 기록합니다.

## 1. 모바일 관리자 페이지 미리보기 UI 통일 및 스크롤 보존
기존에는 리스트에서 상세 화면으로 이동 시 SPA 라우팅 한계로 인해 뒤로가기 시 스크롤이 맨 위로 초기화되는 현상이 있었습니다. 이를 해결하기 위해 전체 관리자 모듈에 오버레이 패턴을 도입했습니다.

*   **공실 관리 (`/m/admin/vacancy`)**: PC 버전의 미리보기를 모바일 전용 `iframe` 오버레이 모달로 대체.
*   **기사 관리 (`/m/admin/article`)**: 기사 제목 및 미리보기 버튼 클릭 시, 화면 전환 없이 리스트 위에 `iframe` 오버레이를 띄우도록 수정하여 스크롤 위치 100% 보존.
*   **히스토리 제어 (`history.pushState`)**: 모달이 열릴 때 브라우저 히스토리를 강제로 추가하고, `popstate` 이벤트를 감지하여 폰의 하드웨어 뒤로가기 버튼을 눌러도 이전 리스트에서 이탈하지 않고 모달만 부드럽게 닫히도록 완벽 구현.

## 2. 모바일 기사 상세보기 렌더링 최적화
모바일 기사 상세 페이지(`m/news/[article_id]/page.tsx`)에 `embed=true` 쿼리 파라미터 처리 로직을 추가했습니다.
*   이를 통해 `iframe` 내부에서 기사가 렌더링될 때는 불필요한 GNB(글로벌 네비게이션 바) 및 헤더가 숨김 처리되어, 팝업창 내에서 군더더기 없는 깔끔한 기사 확인이 가능합니다.

## 3. 고객 관리 뒤로가기(Back Button) 튕김 현상 해결
고객 관리 모듈(`/m/admin/customer`)에서 상세 정보나 등록 화면을 볼 때 상태(`view`)로만 화면을 제어하여 뒤로가기 시 앱 밖으로 튕기는 버그를 해결했습니다.
*   **`goView` 함수 도입**: 상태 전환 시 `window.history.pushState`를 사용하여 히스토리 스택을 남김.
*   이제 상세 화면에서 하드웨어 뒤로가기를 눌러도 안전하게 원래의 고객 리스트 뷰로 복귀합니다.

## 4. 마인 메뉴(GlobalDrawerMenu) 스크롤 위치 완벽 보존
가장 혁신적인 변화로, 우측 상단의 햄버거 메뉴에서 파생되는 모든 링크 이동 시 뒤로가기 스크롤이 풀리던 문제를 완벽하게 해결했습니다.
*   **적용 위치**: `src/app/m/_components/header/GlobalDrawerMenu.tsx`
*   **동작 방식**: 기존의 `<Link>` 태그들을 삭제하고, 클릭 시 넥스트 라우터를 통해 `?menu=open&overlay=[URL]` 파라미터를 추가하도록 변경했습니다.
*   **효과**: 메뉴에서 선택한 모든 페이지(회사소개, 공실뉴스, 내가 등록한 기사 등)가 원본 화면을 덮는 **전체화면 iframe 오버레이**로 열립니다. 따라서 뒤로가기로 팝업을 닫았을 때 백그라운드에 대기하던 기존 리스트의 스크롤 위치가 0.1mm의 오차도 없이 보존됩니다.

## 5. 모바일 공실 상세페이지 기자 홈피 버튼 추가
모바일 환경에서도 등록자가 '기자' 권한을 가졌을 경우 PC와 동일한 시각적 피드백과 동선을 제공하기 위해 UI를 보강했습니다.
*   **적용 위치**: `src/app/m/gongsil/page.tsx`
*   **세부 내용**: 등록자 정보(부동산 이름/등록자 이름) 우측에 `🏠 기자홈피` 핑크색 뱃지 버튼을 추가하여, 클릭 시 해당 기자의 전용 미니 홈페이지(`/m/reporter/[id]`)로 즉시 이동할 수 있게 구현했습니다.
