# AI 부동산 서류 자동 검증 에이전트 연동 가이드

이 문서는 부동산 회원이 신규 가입하거나 정보(사업자등록증, 상호, 대표자 등)를 수정할 때 작동하는 **AI 서류 검증 에이전트(VerifyAgent)** 의 작동 방식 및 트러블슈팅 가이드를 제공합니다.

## 1. 작동 원리
사용자가 사업자등록증 이미지를 업로드하고 **[저장하기]** 를 누르면 다음 프로세스가 진행됩니다.

1. **조건 확인**: 첨부된 사업자등록증 이미지(`biz_cert`) 파일이 폼 데이터에 존재해야만 AI 엔진이 구동됩니다. (텍스트만 수정하고 이미지를 업로드하지 않으면 AI 검증이 스킵됩니다.)
2. **모델 검증**: Google API Key(`최고관리자의 마케팅 정보`에 저장된 키)를 불러와 `gemini-2.5-flash` 모델을 사용하여 이미지를 텍스트로 변환하고 분석합니다.
3. **교차 검증 대상**:
   - `companyName` (상호명): 서류의 텍스트가 입력된 상호명을 포함하고 있는지 (일부 일치 허용)
   - `representative` (대표자명): 서류의 대표자명과 사용자가 입력한 대표자명이 **완벽히 일치**하는지
4. **결과 반영**:
   - **완벽 일치 시**: `status: "APPROVED"` 반환 후 **즉시 정상 승인** 처리됩니다.
   - **불일치 시**: `status: "NEEDS_REVIEW"` 반환 후 상태는 **승인대기(PENDING)** 로 남습니다.
   - **에러 발생 시**: API Key 오류 등 시스템 장애 시 승인대기 처리됩니다.

## 2. 관리자 예외 처리 (수정 사항)
초기에는 `isAdmin === true` 일 때 (최고관리자가 수동으로 회원 정보를 조작할 때) AI 검증을 스킵하도록 설계되었습니다.
하지만 관리자가 직접 테스트를 진행하거나 대행 등록 시에도 AI 자동 승인 혜택을 받을 수 있도록, 현재는 **관리자 화면에서도 무조건 AI 검증이 작동**하도록 조건이 완화되었습니다.

## 3. 거절(보류) 사유 안내 UI
AI가 텍스트 불일치로 승인을 보류할 경우, 사용자가 답답함을 느끼지 않도록 다음 두 가지 피드백이 제공됩니다.

- **저장 즉시 팝업(Alert) 안내**:
  ```text
  🤖 AI 검증 안내: 서류와 입력하신 정보가 일부 불일치하여 관리자 수동 검토(승인대기)로 넘어갑니다.
  [불일치 내역]
  - 상호명 (입력: 주식회사 OO / 서류: 주식회사 OO부동산중개)
  - 대표자 (입력: 홍길동 / 서류: 김철수)
  ```
- **대시보드 영구 노출**:
  DB의 `agencies.rejection_reason` 컬럼에 AI의 불일치 사유가 자동 저장되며, 부동산 회원 대시보드 로그인 시 **[현재 서류 검토가 진행 중입니다]** 노란색 배너 하단에 💡 형태의 점선 박스로 사유가 영구 노출됩니다.

## 4. 자주 발생하는 트러블슈팅
### Q. 모델 404 에러 (models/gemini-1.5-flash is not found)
최근 발급된 구글 API 키는 `gemini-1.5-flash` 모델을 더 이상 지원하지 않습니다. 에이전트 코드는 현재 최신 모델인 **`gemini-2.5-flash`** 로 업그레이드 되어 있습니다. 따라서 `VerifyAgent.ts` 파일에서 모델명이 항상 최신 버전(`gemini-2.5-flash` 이상)으로 유지되어야 합니다.

### Q. 사진은 그대로 두고 텍스트만 고쳐서 저장했는데 AI가 작동하지 않아요.
AI는 서버에 저장된 기존 이미지 URL을 읽는 것이 아니라, 폼에서 신규 업로드되는 File 객체를 가로채서 API로 보냅니다. 따라서 텍스트를 수정한 후 AI 자동 승인을 받고 싶다면, **반드시 사업자등록증 사진을 다시 한 번 업로드(선택)한 후 저장해야** 작동합니다.
