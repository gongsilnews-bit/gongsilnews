# 회원관리 AI 에이전트 기획서 (Membership Agent Spec)

**문서 버전:** 1.0  
**작성일:** 2026-04-24  
**담당 역할:** 부동산 회원의 사업자 등록증 및 중개업 개설등록증 검증 및 가입 자동 승인/반려 프로세스 수행

---

## 1. 개요 (Overview)
기존 최고관리자가 수동으로 진행하던 '부동산 회원(REALTOR)'의 서류 심사 및 가입 승인 절차를 AI 비전 에이전트를 도입하여 자동화합니다. 이 에이전트는 회원가입 시 제출된 이미지 데이터를 읽고 데이터베이스에 입력된 정보와 교차 검증하여 결정을 내립니다.

## 2. 에이전트 작동 흐름 (Workflow)

### Step 1. 트리거 (Event Trigger)
- **방식**: Supabase Webhook 연동
- **조건**: `agencies` 테이블에 새로운 데이터가 Insert 되거나, 기존 회원이 서류를 수정하여 상태가 `PENDING`(승인대기)으로 변경될 때 즉시 발생
- **동작**: Webhook이 AI 에이전트 서버(Next.js API 또는 독립된 AI 서버)로 검토 요청 발송

### Step 2. AI 검증 프로세스 (Data Verification)
1. **서류 판독 (Vision OCR 작업)**:
   - 전송받은 `reg_cert_url`(개설등록증) 및 `biz_cert_url`(사업자등록증) 이미지 링크를 Vision AI 기반(예: GPT-4o Vision, Google Cloud Vision 등)으로 추출
2. **데이터 교차 대조 (Cross-Check)**:
   - AI가 이미지에서 강제 추출한 **상호명**, **대표자명**, **사업자 등록번호**, **부동산 등록번호**를 회원이 폼(UI)에서 기입한 DB 텍스트 정보와 철자 단위로 대조
3. **외부 공공 API 검증 (옵션/추후 도입)**:
   - 국세청 홈택스 사업자상태조회 API를 통해 제출된 사업자번호가 현재 '정상/휴업/폐업' 상태인지 살아있는 데이터로 최종 조회

### Step 3. 승인/반려 판정 및 결과 처리 (Decision & Action)
AI의 신뢰도 및 대조 결과에 따라 아래의 3가지 상태로 자동 분기 처리합니다.

- 🟢 **자동 승인 (APPROVED)**
  - **조건**: 모든 텍스트 정보 일치, 휴폐업 상태 아님 판별 완료 (신뢰도 100%)
  - **Action**: DB의 `agencies.status`를 `APPROVED`로 변경하고, 회원에게 "승인이 완료되었습니다" 알림(이메일/알림톡) 전송.
- 🔴 **자동 반려 (REJECTED)**
  - **조건**: 핵심 정보 불일치 식별 (예: 사업자번호 오류, 남의 서류 이미지 등)
  - **Action**: DB 상태를 `REJECTED`로 설정, "상호명이 일치하지 않으니 다시 서류를 확인해 주세요"와 같은 명확한 사유를 기입하여 재제출 요청 발송.
- 🟡 **보류 및 수동 심사 (FLAGGED / PENDING)**
  - **조건**: 사진 화질 저하로 글씨 인식 실패, 오타 혹은 띄어쓰기 등으로 정보가 애매하게 불일치하여 AI가 판독을 100% 확신할 수 없는 경우.
  - **Action**: 상태를 `PENDING`으로 유지한 채, 최고관리자 대시보드에 **"인적 검토 요망"** 플래그를 달아 관리자가 수동으로 최종 심정을 내리도록 유도 (안정성 보장).

## 3. 기술 스택 제안 (Tech Stack)
- **에이전트 런타임**: `ai_agent` 폴더 내 독립 관리 (Next.js App Router API 형태가 기존 인프라와 결합 시 가장 효율적, 필요 시 Python / FastAPI 환경 분리 고려)
- **이미지 분석**: OpenAI GPT-4o Vision API (자연어 추론을 통한 오타 보정에 유용) 구글 클라우드 Vision API (OCR 속도 측면에 장점)
- **상태 관리**: Supabase Database Webhook & Edge Functions

## 4. 향후 로드맵 (Roadmap)
1. 1단계: **AI Assistant 모드** (실제 승인/반려는 관리자가 클릭하되, AI가 화면에 "일치합니다/불일치합니다" 리포트만 텍스트로 미리 보여줌)
2. 2단계: **Full-Automation 모드** (신뢰도 확보 완료 시, AI가 직접 APPROVED 업데이트 수행)
