# AI 물건보고서(IM Report) 자동화 및 고도화 회의록
**작성일**: 2026년 6월 2일

## 1. 회의 목적
- 브로커(중개인)가 매물 정보를 최소한으로 입력해도, 시스템이 기존 DB(공실정보, 건물대장 등)와 외부 데이터(주변 인프라, 상권 정보 등)를 결합하여 **고품질의 AI 물건보고서(IM)**를 자동으로 완성하도록 시스템을 기획/설계합니다.

---

## 2. 핵심 요구사항 (사용자 피드백)

### A. 기존 공실 정보의 자동 입력 (Zero-Input 지향)
- **Pain Point**: 현재 보고서를 만들 때마다 소재지, 용도지역, 대지/연면적, 건물규모, 주차대수 등을 수기로 입력하거나 폼에서 일일이 수정해야 하는 번거로움이 존재함.
- **Solution**: 매물(Vacancy) ID를 기반으로 이미 등록된 데이터베이스(공실 테이블 및 매물 상세정보)를 불러와 **보고서의 기본 틀(물건 개요, 임대 조건, 건축물 대장 정보 등)**을 100% 자동 채움. 
- **수정/커스텀 가능성**: 자동 채워진 데이터는 EditableBlock 및 좌측 폼을 통해 자유롭게 수정할 수 있도록 유지.

### B. 매물 등록 폼(DB) 입력 항목 확충 (★중요)
- **Pain Point**: 현재 공실 등록 폼(`realty_admin?menu=gongsil&action=write`)에 '단독/다가구' 및 '상가/사무실/건물' 매매 시 꼭 필요한 항목들이 누락되어 있어, 나중에 보고서에서 자동으로 끌어올 수가 없음.
- **Solution (추가 필요 항목)**:
  - **도로 폭**: 건물 가치 평가 및 매매 시 필수 정보 (예: 6m 도로 접)
  - **건물 규모 (총 층수)**: 지하 O층 / 지상 O층 입력란 추가
  - **용도 지역 / 대지면적 / 연면적** (이전 요청사항 포함)
- **기대 효과**: 처음에 한 번만 제대로 등록해두면, 이후 AI 물건보고서 생성 시 이 정보들이 100% 자동으로 들어감.

### C. 주변 정보 (Neighborhood & Infra) 자동 생성
- **Pain Point**: 주변 시세, 인근 상권 분석, 대중교통(지하철역 거리) 등을 브로커가 직접 분석해서 넣기 힘듦.
- **Solution**: 매물의 주소(좌표)를 바탕으로,
  - 반경 내 지하철/대중교통 정보 자동 추출 (Connectivity)
  - 반경 내 상권 데이터, 주변 시세 데이터 API 연동 혹은 기존 DB 활용 
  - AI(Gemini)를 활용하여 주변 입지에 대한 "Strategic Advisory(전문가 소견)" 또는 "입지 분석 코멘트" 자동 초안 작성

## 3. AI 물건보고서 기본 틀 (Draft Architecture)

아래는 브로커에게 제공될 **자동화된 5~6페이지 구성** 제안입니다. (현재 구조를 바탕으로 발전)

| 페이지 | 메뉴명 (주제) | 자동 채움(Auto-Fill) 데이터 소스 | AI / 외부 데이터 역할 |
| :--- | :--- | :--- | :--- |
| **Page 1** | **보고서 표지 & 물건 개요** (Overview) | 주소, 대지/연면적, 건축연도, 용도, 주차장, 승강기 유무 (기존 DB) | 브로커의 코멘트 없이도 깔끔한 요약 테이블 자동 렌더링 |
| **Page 2** | **가격 및 가치 평가** (Valuation) | 기존 희망가, 현재 급매가, 탁상감정가 등 등록된 수치 반영 | AI가 주변 시세 데이터와 비교하여 **'투자가치 요약문'** 자동 작성 |
| **Page 3** | **임대차 현황** (Rent Status) | 해당 매물의 층별 임대 현황 (보증금/월세/관리비 등 DB 연동) | 총 보증금, 총 월세액, 예상 수익률(Yield) 자동 계산 및 표기 |
| **Page 4** | **주변 입지 및 인프라** (Location) | 주소 기반 좌표 (Kakao Map 연동) | 반경 N km 내 핵심 앵커테넌트, 지하철역 도보 시간 등 크롤링/API 연동 |
| **Page 5** | **현장 사진 모아보기** (Photos) | 매물 등록 시 첨부한 사진들 (메인, 서브 등) 최대 4~6장 자동 배치 | (선택적) AI가 사진의 특징을 분석해 짧은 캡션 생성 |
| **Page 6** | **브로커 & 회사 정보** (Contact) | 로그인한 중개인의 프로필 정보 (이름, 직급, 연락처) | 중개사무소 로고 및 기본 소개말 자동 연동 |

## 4. 논의 및 결정 필요 사항 (Next Steps)

회의를 통해 다음 사항들을 결정하고 싶습니다.

1. **자동화 우선순위**
   - 현재 등록된 정보 중 "가장 매번 반복해서 적어야 했던 것"은 무엇인가요? (예: 임대 현황 표, 건축물 대장 표 등)
2. **AI의 개입 수준**
   - 가격/투자가치 분석 시, AI가 자동으로 글(Advisory)을 적어주는 기능을 바로 적용할까요? 아니면 브로커가 선택한 키워드만으로 조합되게 할까요?
3. **주변 정보 연동 방식**
   - 주변 정보(지하철, 학군, 상권 등)를 가져올 때, 현재 사용 중인 다른 API나 데이터 소스가 있다면 그것을 우선 연동할까요?
