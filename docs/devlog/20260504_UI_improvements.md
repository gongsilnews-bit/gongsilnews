# 2026-05-04 UI/UX 개선 및 버그 수정 작업 내역

## 1. 기자 프로필 카드 최적화 (PC & Mobile 공통)
* **디자인 및 정보 동기화**: PC와 모바일 환경의 기자 프로필 카드 레이아웃을 통일했습니다. 
  * 대표자 이름, 중개사무소 등록번호 추가 노출.
  * 일반 사무실 번호와 스마트폰 번호를 구분하여 가독성 높게 수직 배치.
* **SNS 링크 연동 버그 수정**: PC 환경에서 대표 SNS가 노출되지 않던 문제를 수정했습니다. (`agencyInfo.sns_links` 대신 `profile.sns_links`를 참조하도록 수정)
* **액션 버튼 개선**: 
  * "오시는길" 및 "공유" 버튼의 기존 촌스러운 이모지(📍, 🔗)를 깔끔한 흰색 선형 SVG 픽토그램으로 교체하여 전문성을 높였습니다.
  * "오시는길" 클릭 시 기존 네이버 지도 대신 **다음 지도(Kakao Map) 검색 페이지**로 바로 연결되도록 링크(map.kakao.com/link/search)를 수정했습니다.

## 2. 댓글 더보기 기능 추가 (NewsReadContent)
* 기사 본문 하단의 댓글 영역에서, 로딩 시 전체 댓글을 한 번에 보여주지 않고 **기본적으로 최근 댓글 5개까지만 노출**되도록 최적화했습니다.
* 댓글이 5개를 초과할 경우 하단에 **"댓글 더보기 (N개)" 버튼**이 렌더링됩니다.
* 버튼 클릭 시 숨겨져 있던 나머지 댓글 전체가 부드럽게 펼쳐집니다.

## 3. PC 헤더 검색 기능 고도화 (최근 검색어 레이어)
* PC 우측 상단의 돋보기 아이콘 클릭 시, 모바일이나 네이버처럼 검색어 입력창 아래에 **"최근 검색어" 드롭다운 레이어**가 나타나도록 구현했습니다.
* **기능 상세**:
  * LocalStorage(`gongsil_recent_searches`)를 사용하여 최대 10개의 최근 검색어를 기록하고 노출.
  * 목록에서 이전 검색어 클릭 시 즉시 해당 키워드로 재검색.
  * 개별 검색어 삭제 (X 버튼) 및 "전체 삭제" 기능 지원.
  * 검색창 외부 영역 클릭 시 드롭다운 자동 닫힘 처리.
* **호환성 에러 패치**: 기존 모바일 환경 등에서 최근 검색어를 객체 배열(예: `[{term: "검색어", date: "..."}]`) 형태로 저장하던 데이터 구조 때문에 발생했던 React 렌더링 에러를 해결했습니다. 데이터를 불러올 때 자동으로 문자열(`string`)만 추출하도록 정규화 로직을 추가하여 오류를 완벽히 차단했습니다.
