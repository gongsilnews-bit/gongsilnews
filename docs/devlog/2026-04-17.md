### 13. 우측 사이드바(많이 본 뉴스 및 추천 매물) 최적화
- `NewsListLayout` 사이드바의 하드코딩된 '추천 공실' 제거 
- 개별 카테고리(정치, 부동산, 세무 등) 페이지에서 `많이 본 뉴스`가 글로벌 통계가 아닌 **해당 카테고리의 조회수 랭킹**만을 가져오도록 설정 (`getArticles({ section2: ... })` 쿼리 수정)
- 우측 사이드바의 '많이 본 뉴스' 위젯 제목을 `{현재 카테고리명} 많이 본 뉴스`로 동적으로 변경 (예: "부동산·주식·재테크 많이 본 뉴스")

### 14. 기사 상세화면 알림 UI 및 기타 카테고리 로직 개선
- 사용자 경험을 위해 브라우저 기본 `alert` 함수 호출을 찜하기 알림처럼 부드러운 하단 팝업 형태인 블랙 커스텀 토스트로 모두 교체
- `기타 전체보기` 카테고리에서 아무 기사도 나타나지 않던 이슈 해결
- `article.ts`의 `getArticles` 액션에서 배열 형태의 `section2` 필터링 지원 (`string[]`)
- 기타 카테고리 클릭 시 IT/가전/가구, 스포츠, 인물/기타의 3가지 서브 카테고리를 합쳐서 한 번에 출력되도록 구현
- 캡쳐해 주신 기사 상세 하단 부분에 `사진이 없는 부동산 회원의 매물도 리스트에 모두 표시되도록` 사진 필터링 제한조건을 해제
- 로그인 유도 시 사용자 친화적인 경험을 위해 확인 모달 UI(`네, 아니요`)를 블랙 테마로 커스터마이징하여 모든 댓글, 좋아요, 리액션, 입력 칸 클릭 인터렉션에 적용
- 부동산 회원의 추천 매물이 사진이 없을 시 노출되던 회색 더미(Dummy) 박스 이미지 레이아웃 렌더링 제거
