# 2026-05-16 개발 일지: 모바일 UX 최적화 및 정식오픈 예외 처리

## 1. 사진 업로드 최적화 및 서버 용량 이슈 해결
- **문제**: 모바일 환경에서 다수의 고해상도 사진 업로드 시 서버 용량 제한(Payload Too Large) 초과로 인한 에러 발생.
- **해결 방안**:
  - 클라이언트 단에서 브라우저 `canvas`를 활용해 이미지를 **WebP 포맷으로 자동 압축** (최대 해상도 1920x1080)하도록 로직 적용 (`compressToWebP`).
  - `next.config.ts` 파일 내 `experimental.serverActions.bodySizeLimit` 속성을 `10mb`로 상향 조정하여 근본적인 전송 실패 문제 해결.

## 2. 모바일 공실 등록 폼: 층수 입력 UX 개선
- **문제**: 반지하, 옥탑, 전체층 등 특수한 층수를 입력할 때 모바일 키보드 조작이 번거로움. 또한 다른 일반 층(예: 10층, 13층)의 경우 어떻게 입력해야 할지 혼란 야기.
- **해결 방안**:
  - '해당층' 입력칸 라벨을 `해당층 (직접입력)`으로 변경하여 숫자 입력이 가능함을 명확히 안내.
  - 하단에 퀵 버튼 (칩 UI) 추가: `[반지하]`, `[옥탑]`, `[지하 1층]`, `[지하 2층]`, `[전체층]`. 버튼 클릭 시 해당 값이 인풋에 자동 입력되도록 구성하여 모바일 접근성 및 편의성 극대화.

## 3. 정식 오픈(6월 1일) 전 가림막 예외 처리 (권한 제어)
- **문제**: 6월 1일 정식 오픈 전까지 실시간 공실 지도 열람을 막는 가림막(오버레이)이 일괄 적용되어 있어, 매물을 등록한 당사자조차 자신의 매물을 미리보기 할 수 없는 문제 발생.
- **해결 방안**:
  - `GongsilClient.tsx` (PC 맵) 및 `/m/gongsil/page.tsx` (모바일 맵) 수정.
  - **최고 관리자(ADMIN)** 계정인 경우, 또는 **접속한 사용자(currentUser)가 해당 매물의 등록자(owner_id)와 일치**하는 경우에는 가림막이 보이지 않고 정상적으로 매물 상세 페이지를 열람할 수 있도록 예외 로직 추가.

## 4. 모바일 관리자 공실 목록 UI 개선 (물건번호 강조)
- **문제**: 공실 관리 목록에서 물건번호가 우측 날짜 옆에 작게 `#28` 형태로 표시되어 가독성이 떨어짐.
- **해결 방안**:
  - `/m/admin/vacancy/page.tsx` 목록 아이템 렌더링 영역 수정.
  - 물건번호를 좌측 물건 종류(예: 오피스텔, 사무실) 바로 우측으로 이동시키고, 텍스트 형태를 `No.28`과 같이 변경한 후 빨간색(`#ef4444`)을 적용하여 시각적으로 쉽게 인지되도록 강조.

## 5. 공실 등록 완료 후 뒤로가기(Back) UX 개선
- **문제**: 모바일 공실 등록 화면에서 등록을 완료하면 목록(`/m/admin/vacancy`)으로 이동(`router.push`)하는데, 이후 디바이스의 뒤로가기 버튼을 누르면 이전에 작성 완료한(또는 빈) 등록 폼으로 다시 진입하는 현상 발생.
- **해결 방안**:
  - 등록 성공 및 뒤로가기(취소) 시 동작하는 로직을 `router.push()`에서 **`router.replace()`**로 변경.
  - 방문 기록(History)을 덮어씌움으로써, 목록에서 뒤로가기 클릭 시 불필요하게 폼으로 돌아가는 현상을 방지하고, 직방·다방과 같은 깔끔한 네비게이션 흐름을 제공.
