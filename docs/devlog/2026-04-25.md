# 2026-04-25 개발 일지 (Devlog)

## 1. 메인 페이지 & 홈페이지 매물 리스트 회원별 마스킹 적용

### 문제점
- 공실열람(`/gongsil`)의 `GongsilClient.tsx`에는 회원 등급별 매물 마스킹이 구현되어 있었으나, **메인 페이지**("실시간 공실" 섹션)와 **홈페이지** 매물 리스트에는 마스킹이 적용되지 않아 비회원/일반회원에게도 `부동산노출` 전용 매물의 주소, 중개보수, 연락처가 그대로 노출되는 상태였음.

### 해결책
`GongsilClient.tsx`(라인 1850)에 이미 구현된 마스킹 로직을 2개 파일에 동일하게 적용:

```tsx
const isMasked = prop.exposure_type === '부동산노출' && userLevel < 2;
const showCommission = userLevel >= 2;
```

### 수정 파일

#### `src/components/home/HeroMapSection.tsx` (메인 페이지 "실시간 공실")
- `getPermissionLevel` 유틸 import 및 Supabase 유저 인증 상태 감지 로직 추가
- `userLevel` / `isAuthModalOpen` 상태 추가
- 리스트 카드 렌더링에 `isMasked` / `showCommission` 판별 적용:
  - 마스킹 시 주소를 `XXX`로 치환
  - 중개보수/연락처 숨김
  - 클릭 시 `AuthModal` 표시 (회원가입 유도)
  - "🔒 부동산회원 가입 시 무료 열람" 배지 표시
- `AuthModal` 컴포넌트 렌더링 추가

#### `src/app/(main)/homepage/page.tsx` (홈페이지 매물 리스트)
- 동일한 유저 인증 + 권한 레벨 감지 추가
- 리스트 카드에 마스킹 적용 (주소 X 치환, 중개보수 숨김, AuthModal 연동)
- `AuthModal` 컴포넌트 렌더링 추가

## 2. 회원별 리스트 현황 정리

플랫폼 전체 페이지에 대한 회원 등급(0~5단계)별 접근 권한 매트릭스를 정리.

### 회원 등급 체계

| 레벨 | 등급명 | 판별 조건 |
|:---:|---|---|
| 0 | 비회원 | 로그인 안 한 상태 |
| 1 | 일반회원 | `role = USER` |
| 2 | 무료부동산회원 | `role = REALTOR`, `plan_type` 없음 |
| 3 | 공실뉴스부동산 | `role = REALTOR`, `plan_type = news_premium` |
| 4 | 공실등록부동산 | `role = REALTOR`, `plan_type = vacancy_premium` |
| 5 | 최고관리자 | `role = ADMIN` |

권한 레벨 산정은 `src/utils/permissionCheck.ts`의 `getPermissionLevel()` 함수에서 중앙 관리.
