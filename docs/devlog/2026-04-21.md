# 2026-04-21 개발일지

## ✅ 완료 항목

### 1. 부동산 회원 다중 요금제 (Membership Tiers) 설계 및 DB 반영
- `members` 테이블에 부동산 회원의 이용 한도를 제어하기 위한 컬럼 추가:
  - `max_vacancies` (공실 매물 최대 등록 가능 건수, 기본 5)
  - `max_articles_per_month` (월 최대 기사 작성 가능 건수)
  - `current_month_articles_count` (당월 기사 작성 수)
- 부동산 회원 요금제 구분을 위한 컬럼 추가 반영 (수동 SQL 스크립트 작성 - `sql/update_members_plan.sql`):
  - `plan_type` (무료: free, 공실뉴스부동산: news_premium, 공실등록부동산: vacancy_premium - 기본값 'free')
  - `plan_start_date` (요금제 적용 시작일)
  - `plan_end_date` (요금제 적용 종료일)

### 2. 관리자 페이지 내 부동산 요금제 관리 UI 구현
- **회원 수정 폼 (`MemberRegisterForm.tsx`)**
  - 관리자가 '부동산 회원'으로 회원을 구별/수정할 때, 새롭게 **[부동산 요금제]** 드롭다운이 렌더링되게 구현.
  - 무료가 아닌 유료(`news_premium`, `vacancy_premium`) 플랜을 선택할 경우, 요금제 적용 기간(시작일~종료일)을 설정할 수 있는 달력 `input` 활성화.
  - Supabase `actions.ts` 업데이트를 통해 변경된 요금제 및 기간 값이 DB에 안전하게 저장되도록 로직 수정 (`adminCreateMember`, `adminUpdateMember`).
- **회원 목록 (`MemberSection.tsx`)**
  - 회원 목록의 '회원구분' 표기란에, 유료 회원일 경우 요금제 명칭이 직관적으로 표기되도록 수정. 
  - (예: `부동산회원 (공실뉴스)`, `부동산회원 (공실등록)`)

### 3. 로그인 및 유지보수 모드 (Maintenance Mode) 트러블슈팅
- **실서버 접근 락 해제 안내**
  - 이전 작업에서 적용해두었던 Vercel의 환경변수 `NEXT_PUBLIC_MAINTENANCE_MODE=true`로 인해 일반 접속/리다이렉트가 막히는 "사이트 개편중" 이슈 트러블슈팅. 
  - Vercel 대시보드에서 해당 값을 `false`로 수정 및 Redeploy 하도록 안내.
- **로컬 보안 인증(PKCE) 충돌 해결**
  - 카카오 로그인 시도 시 `gongsilnews.com` 실서버로 리다이렉트 되어버려 로컬 관리자 권한이 튕기는 현상 발생. 
  - Supabase 세팅 내 `Authentication` > `URL Configuration` > `Additional Redirect URLs`에 로컬(`localhost:3000`)을 등록하여 개발 환경에서도 안전하게 사이트 주소별 독립적인 인증이 가능하도록 해결 방법 가이드.
