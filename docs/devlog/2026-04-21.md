  - `plan_end_date` (요금제 적용 종료일)

### 2. 관리자 페이지 내 부동산 요금제 관리 UI 구현
- **회원 수정 폼 (`MemberRegisterForm.tsx`)**
  - 관리자가 '부동산 회원'으로 회원을 구별/수정할 때, 새롭게 **[부동산 요금제]** 드롭다운이 렌더링되게 구현.
  - 무료가 아닌 유료(`news_premium`, `vacancy_premium`) 플랜을 선택할 경우, 요금제 적용 기간(시작일~종료일)을 설정할 수 있는 달력 `input` 활성화.
  - Supabase `actions.ts` 업데이트를 통해 변경된 요금제 및 기간 값이 DB에 안전하게 저장되도록 로직 수정 (`adminCreateMember`, `adminUpdateMember`).
  - (예: `부동산회원 (공실뉴스)`, `부동산회원 (공실등록)`)

### 5. 회원 목록 리스트 구성 직관화 및 자동 상태 전환
- **데이터 중심 리스트 개편**: 아이디/이름 등 기본 정보를 좌측으로 밀고, 우측에 **공실 현황(현재/한도), 기사 현황(월간/한도), 홈페이지 ID, 만료 종료일**을 한눈에 볼 수 있도록 칼럼 배치.
- **승인 상태 자동 최신화**: '정상승인' 처리 후 7일(1주일)이 경과하면 리스트 상에서 '정상' 텍스트로 자동 변경되도록 하여 신규 회원과 안착 회원의 시각적 구분 기능 제공.

### 3. 로그인 및 유지보수 모드 (Maintenance Mode) 트러블슈팅
- **실서버 접근 락 해제 안내**
  - 이전 작업에서 적용해두었던 Vercel의 환경변수 `NEXT_PUBLIC_MAINTENANCE_MODE=true`로 인해 일반 접속/리다이렉트가 막히는 "사이트 개편중" 이슈 트러블슈팅. 
  - Vercel 대시보드에서 해당 값을 `false`로 수정 및 Redeploy 하도록 안내.
- **로컬 보안 인증(PKCE) 충돌 해결**
  - 카카오 로그인 시도 시 `gongsilnews.com` 실서버로 리다이렉트 되어버려 로컬 관리자 권한이 튕기는 현상 발생. 
  - Supabase 세팅 내 `Authentication` > `URL Configuration` > `Additional Redirect URLs`에 로컬(`localhost:3000`)을 등록하여 개발 환경에서도 안전하게 사이트 주소별 독립적인 인증이 가능하도록 해결 방법 가이드.

### 6. 부동산 홈페이지 설정 폼 UI 구현 (SaaS 멀티테넌트 연동)
- **배경**: 템플릿 서버(Port 3001)와 공실뉴스 어드민(Port 3000)을 연동하여, 각 부동산 중개사에게 독립 서브도메인 홈페이지(`xxx.gongsilnews.com`)를 제공하는 SaaS 구조.
- **Server Action 신규 생성 (`src/app/actions/homepage.ts`)**:
  - `getHomepageSettings(ownerId)` — 현재 설정 조회 (없으면 null)
  - `saveHomepageSettings(ownerId, data)` — upsert 방식 저장 (owner_id 기준)
  - `checkSubdomainAvailable(subdomain, ownerId?)` — 서브도메인 중복 검사
  - `uploadHomepageFile(formData)` — 로고/파비콘 이미지를 Supabase Storage에 업로드
  - `ensureBucket()` — `homepage_assets` 버킷이 없으면 자동 생성
- **HomepageSection 컴포넌트 (`src/components/admin/sections/HomepageSection.tsx`)**:
  - 8개 폼 필드: 서브도메인 주소, 템플릿 선택(오피스형/아파트형), 로고 업로드, 파비콘 업로드, 사이트 제목, 대표 연락처, 회사 소개글, 홈페이지 활성화 토글
  - 서브도메인 실시간 중복 검사 (600ms 디바운스)
  - **요금제별 기능 제한**: `free` → 템플릿/로고 변경 잠금 + 안내 배너 표시
  - **WebP 압축**: 로고(최대 400px), 파비콘(최대 128px) — Canvas API 기반 클라이언트 측 압축 후 `.webp`로 업로드
- **realty_admin 페이지 연결 (`src/app/realty_admin/page.tsx`)**:
  - `HomepageSection` lazy import 추가
  - `plan_type` 조회 추가 (`members` 테이블에서 함께 fetch)
  - 기존 🚧 플레이스홀더 → 실제 `HomepageSection` 컴포넌트로 교체
- **Supabase 연동 테이블**: `homepage_settings` (이미 생성됨 — 핸드오프 문서 참조)
