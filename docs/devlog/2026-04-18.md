### 6. 메인 페이지 로딩 속도 최적화 (3단계)
- **문제**: Vercel 프로덕션에서 메인 페이지 첫 로딩이 체감상 느림
- **원인 분석**:
  - `CategoryNewsGrid`에서 카테고리별 8개의 개별 DB 쿼리 실행
  - `layout.tsx` → `page.tsx` → `CategoryNewsGrid` 순차 실행 (직렬 대기)
  - 캐시 만료 시간이 60초로 짧아 빈번한 갱신
- **최적화 1: 쿼리 통합** (DB 왕복 75% 감소)
  - 기사 8개 개별 쿼리 → 2개 벌크 쿼리(`뉴스/칼럼` 전체 100건 + `우리동네부동산` 30건)
  - `CategoryNewsGrid`를 순수 렌더 컴포넌트로 변환 (자체 fetch 제거, props로 데이터 수신)
  - JS에서 `section2` 기준으로 카테고리별 분류 (`filter` + `slice`)
- **최적화 2: 병렬 요청** (직렬 대기 제거)
  - `page.tsx`에서 배너(3개) + 공실 + 특강 + 기사(2개) = 총 7개 요청을 단일 `Promise.all`로 병렬 실행
  - `CategoryNewsGrid`가 더 이상 서버 컴포넌트가 아닌 일반 컴포넌트로 전환 → layout/page와의 직렬 대기 제거
- **최적화 3: 캐시 확장** (히트율 5배 향상)
  - `page.tsx`의 `revalidate`: 60초 → 300초 (5분)
  - `article.ts`의 `unstable_cache` revalidate: 60초 → 300초
  - 기사 작성/수정 시 `revalidateTag("articles")`로 즉시 갱신되므로 데이터 신선도 유지
- **추가 권장**: Vercel Pro($20/월) + 도쿄 리전 전환 시 한국↔서버 왕복 200ms → 30ms로 극적 개선 가능
