# 07. 프론트엔드 모듈화 및 성능 최적화 전략 (Frontend Component & Performance Strategy)

## 1. 개요 (Overview)
현재 **Next.js (App Router) + Tailwind CSS + Vercel + Supabase** 스택을 바탕으로 구축 중인 공실뉴스 포털은 매우 방대한 기사 및 매물 데이터를 빠르고 안정적으로 서비스하기 위한 최상위 인프라를 갖추고 있습니다.
초대용량 트래픽이 발생하더라도 쾌적하게 동작하도록 하기 위한 핵심 아키텍처 규칙과 "레고 블록(Component)" 기반의 개발 방법론을 정의합니다.

## 2. 성능 최적화 핵심 3원칙 (Performance Rules)

### ① 데이터 캐싱과 렌더링 무효화 (ISR & Revalidation)
* **문제:** 모든 사용자가 접속할 때마다 DB에서 뉴스나 매물 데이터를 실시간으로 가져오면 데이터베이스 과부하 및 렌더링 지연이 발생합니다.
* **해결책:** Next.js의 정적 생성(SSG)과 점진적 재생성(ISR)을 활용합니다. 특정 페이지의 만료 시간 설정(Time-based Revalidation) 또는 관리자가 기사/매물 등록/수정 시에만 화면을 캐시 무효화(On-demand Revalidation) 처리하여 서버 비용을 줄이고 응답 속도를 극대화합니다.

### ② 이미지 자동 최적화 (Next.js `<Image>`)
* **문제:** 고해상도 매물 사진과 기사 배너 이미지는 웹 브라우저 로딩 지연의 1순위 원인입니다.
* **해결책:** 기존 HTML의 `<img>` 태그 사용을 엄격히 통제하고, Next.js의 `next/image` 컴포넌트를 사용합니다. 이를 통해 사용자 기기 사이즈에 맞춘 해상도 리사이징, WebP/AVIF 포맷 자동 변환 및 지연 로딩(Lazy Loading)이 자동 적용되도록 합니다.

### ③ 데이터 분할 로딩 방식 (Pagination & Infinite Scroll)
* **문제:** 수만 개의 매물이나 뉴스 목록을 한 번에 렌더링하면 브라우저 메모리 초과 현상이 발생합니다.
* **해결책:** Supabase의 `limit()` 및 `range()` 쿼리를 적극 활용하여 한 번에 20개 내외의 필수 데이터만 로드하며, 사용자의 스크롤이나 더보기 버튼을 통해 상태(State) 기반 지연 로딩을 설계합니다.


## 3. "레고 블록" 컴포넌트 모듈화 전략 (Component Modularization)
유지보수성을 높이고 개발 속도를 2배 이상 당기기 위해 모든 UI 요소를 독립적인 블록(Component)으로 분리합니다.

### 필수 공용 컴포넌트 (Shared UI Blocks)
1. **`StatusBadge` (상태 뱃지 컴포넌트)**
   * **용도:** `[진행중]`, `[핫이슈]`, `[전세/매매]` 등 반복 사용되는 라벨.
   * **특징:** 배경색과 글자색의 일관성을 타입 매개변수(`type="hot"`) 하나만으로 통제 가능하게 설계.
2. **`Card` (정보 노출용 범용 카드 컴포넌트)**
   * **용도:** 메인 뉴스 홈, 마이페이지, 우리동네 뉴스 리스트.
   * **변형:** `<ArticleCard>`(뉴스 전용), `<PropertyCard>`(부동산 레이아웃 전용).
   * **특징:** 제목, 썸네일, 날짜, 작성자 등의 구조적 정렬을 담당.
3. **`SkeletonUI` (로딩용 스켈레톤 컴포넌트)**
   * **용도:** 데이터 Fetching 시 빈 화면 대신 표시하여 UX 체감 속도를 높임.
   * **특징:** 회색톤 박스에 `animate-pulse` 테일윈드 클래스를 사용하여 모던하게 디자인.
4. **`FormInput` (입력 폼 필드 컴포넌트)**
   * **용도:** 관리자 글쓰기 폼, 로그인, 회원가입 모달 등.
   * **특징:** Validation(에러 검증), 레이블 렌더링, 빨간색 에러 메시지 표기를 폼 내부 모듈 안에서 한 번에 통제.


## 4. 서버 vs 클라이언트 컴포넌트 구획화
* **서버 컴포넌트 (Server Components)**
  * **조건:** 클릭 이벤트나 타이머 등 화면 내 상태(State)를 변경하지 않는 순수한 보여주기 블록. (예: 뉴스 기사 본문, 카테고리 헤더)
  * **목적:** 자바스크립트 번들 사이즈를 대폭 줄여 클라이언트 렌더링 부하 최소화.
* **클라이언트 컴포넌트 (Client Components - `use client`)**
  * **조건:** '좋아요', '찜하기 끄기', '다크모드 토글', '폼 입력' 등 인터랙티브 블록.
  * **목적:** 부모 계층에서 무분별한 `use client` 선언을 막고, 버튼처럼 꼭 필요한 잎사귀 계층(Leaf Node)에서만 사용하도록 컴포넌트 조망 설계.
