# 💡 IDEA 03: 공실열람 (Vacancy Map) 아키텍처 및 UX 고도화 설계안

본 문서는 **공실열람(프롭테크 지도 매물 서비스)** 영역 개발을 위해 진행된 기획 회의 내용을 종합하여 Next.js 및 Tailwind 환경에 맞춰 기술적/시각적 구현 전략을 명문화한 것입니다.

> **🔥 코어 철학: "부동산 중개사(B2B)에게 제공하는 궁극의 전문가용 플랫폼"**
> 단순 전/월세 검색을 넘어, 하루 종일 화면을 띄워두고 치열하게 상권과 매물을 분석하는 **현업 공인중개사들의 '업무 최적화'**에 초점을 맞춘 킬러(Killer) 시스템입니다. UI 인터페이스의 유연도를 극대화하여 중개사의 업무 방식에 지도 플랫폼이 완벽히 맞춰지게 세팅합니다.

---

## 1. 전용 독립 뷰포트 확보 (Full-Screen Dedicated App)
기존의 **글로벌 헤더 및 푸터는 과감히 삭제**하거나 숨김(`display: none`) 처리하여, 네이버 지도나 직방과 같은 **화면 전유형(Full-viewport) 웹 애플리케이션** 형태를 띱니다.
- **공실열람 전용 특화 헤더 장착**: 공실뉴스 로고(집 모양)와 아파트/오피스텔 등 핵심 매물 카테고리 탭이 포함된 커스텀 슬림 헤더를 최상단에 고정합니다.
- **챗봇 배제**: UI 시야를 심각하게 가릴 수 있는 우측 하단 플로팅 챗봇 아이콘은 구현하지 않고 지도 영역을 최대로 확보합니다.

## 2. 0초 반응형 필터 체계 (Zero-Latency Dynamic Filtering)
사용자가 상단의 아파트/상가/빌라 탭을 누르면, **새로고침 없이 하위 검색 조건바가 종류별로 즉각 교체**됩니다.
- **조건바 모듈화**: `<AptFilter />`, `<CommercialFilter />` 등 카테고리별 조건 컴포넌트를 미리 브라우저(Client State) 단위로 불러두어 탭 스위칭 0.001초의 무지연 경험을 선사합니다.

## 3. 3단계 다중 정밀 탐색 모드 (B2B 전문가용 업무 스위칭)
부동산 중개사들의 저마다 다른 업무 스타일(핀포인트 위주, 거시적 상권 분석 위주, 빠른 리스트 스캔 위주)을 모두 만족시키기 위하여, 데이터 리로딩(Re-fetching) 없이 3가지 형태의 UI 레이아웃을 0초의 딜레이로 즉각 스위칭할 수 있는 툴바 옵션 `[ 📍지도 | 📋리스트 | 🟩블럭 ]` 을 제공합니다.
- **① 지도 뷰 (Map View)**: 줌 확대 및 '점 단위'의 정밀한 핀(마커)과 클러스터링으로 특정 골목이나 주소지 코앞의 매물을 세밀하고 정교하게 찾아내어 고객에게 브리핑할 때 씁니다.
- **② 리스트 뷰 (List View)**: 지도를 완전히 배제하고 매물 리스트를 위아래로 길게 뿌립니다. "특정 조건의 리스트"를 가장 빠른 속도로 광범위하게 스캔하고 텍스트 위주의 서류 작업을 진행할 때 최적화된 모드입니다.
- **③ 블럭 지도 뷰 (Block Map View)**: ②번 리스트 뷰 상단에 법정동/행정동 단위의 **'도형(폴리곤 블럭)'**으로 구분된 지도가 제공됩니다. 화면에서 특정 동네 블럭을 클릭하면 파란색으로 선택되며(스크린샷 참조), 그 아래로 해당 블럭 안의 매물들만 쫙 필터링되어 출력됩니다. 거시적인 '동 단위' 상권 분석 흐름 및 지역 브리핑에 특화된 B2B 전용 모드입니다.

> **⚡ [기술 검토] 3가지 뷰를 무손실 스위칭할 때의 속도 저하 방어 전략**
> 3개의 무거운 UI 템플릿이 한 페이지에 존재하면 초기 로딩이 느려질 수 있다는 우려를 완벽히 불식시키기 위해 **Next.js의 동적 지연 로딩(Dynamic Import & Code Splitting)** 기술을 적용합니다. 
> 사용자가 접속한 최초의 뷰(예: 지도 뷰)에 필요한 코드 33%만 즉시 다운로드되고, 나머지 리스트 뷰와 블럭 뷰의 코드는 다운로드되지 않고 대기합니다. 사용자가 뷰 스위치 버튼을 클릭하는 찰나의 순간에만 가벼운 조각 코드를 불러와 변신시키므로, 복잡한 기능이 추가되어도 첫 접속 렌더링 속도(TTI)는 극한으로 유지됩니다.

## 4. 양방향 맵-리스트 동기화 (Bi-directional Sync)
- **클러스터 클릭 필터링**: 우측 지도에서 매물이 뭉쳐있는 마커 (예: `파란색 [9]`)를 클릭하면, **좌측 리스트의 헤더가 "선택된 클러스터 9개"로 변경되며 리스트가 해당 9개 매물로 즉시 필터링**됩니다.
- 리스트 내부 항목 하나를 짚으면(Hover), 지도 쪽 해당 마커가 통통 튀는 등의 반응형 하이라이트가 동반됩니다.

## 5. 상세 보기 하이브리드 아키텍처 전략
매물 버튼을 눌렀을 때의 동작입니다.
- **마스터 페이지 원칙**: `gongsil/[id]` 라는 독립된 마스터 상세보기 템플릿(URL)은 무조건 하나 존재합니다.
- **지도 뷰 기반 (PC)**: 페이지가 넘어가지 않고 마스터 콘텐츠가 지도 우측에서 **가림막 형태의 슬라이딩 사이드 패널(Sliding Panel)** 위치에 즉시 튀어나옵니다. URL Query만 조작하여 카톡 공유(Deep Link)를 문제없이 지원합니다.
- **리스트 뷰 기반 (PC 정통파)**: 리스트 모드일 때는 클릭 시 **새 탭(`target="_blank"`)**으로 열어주어, 유저가 여러 매물 창을 띄워놓고 스펙을 비교(크로스오버) 할 수 있도록 합니다.
- **모바일 환경 (Mobile Modal)**: 모바일에서는 클릭 시 화면 전체 100%를 우측에서 좌측으로 덮어버리는 **Full-Screen Slide Modal** 형식으로 매물 디테일을 전개하며, 최상단 좌측에 큰 `< 뒤로가기` 버튼을 제공합니다.

## 6. 매물 상세 패널 내부 UI/UX 규정
- **사진 갤러리 (크롭 & 라이트박스)**:
  - 정보가 묻히는 것을 방지하기 위해 좁은 패널 안에서는 와이드하게 잘라서 보여주는 **컴팩트 크롭 뷰(높이 240px 제한)** 를 사용합니다.
  - 사용자가 사진을 클릭하면, 페이지 일대가 암전되며 전체화면으로 **고화질 캐러셀(Lightbox) 사진 슬라이드**가 열립니다. (`1/15` 페이징 포함)
  - 등록된 사진이 0장이라면 **사진 영역을 완전히 지워버리고 설명 텍스트를 최정상으로 끌어올려** 쓸데없는 여백을 없앱니다.
- **등록자(중개사) 상세 정보 탭**: 
  - 서브 탭을 두어 [해당 중개사가 올린 타 매물 리스트(매매/전세 분리)] 현황을 노출합니다. 여기서 다른 매물을 클릭해도 슬라이딩 패널 안에서 끊김 없이 뷰포트만 자연스레 교체됩니다.
  - **평가 및 댓글창**: 하단에 유저 소통/리뷰용으로 기존 뉴스 기사에서 활용했던 모듈을 이식합니다.
- **Sticky Call To Action (전화 연결)**: 스크롤이 아무리 내려가도 가장 하단에 **[가격 + 파란색 '연락처 보기' 버튼]**이 고정되어 즉시 연락을 유도합니다.

---
**설계 결론:**
모든 액션의 중심은 `새로고침 없는 즉각적인 뷰포트 스왑(React State Base)` 이며, 이로 인해 서버 데이터베이스의 부하를 극도로 줄이면서 네이티브 모바일 애플리케이션에 필적하는 UX(사용성)를 산출해 내는 것이 목표입니다.
