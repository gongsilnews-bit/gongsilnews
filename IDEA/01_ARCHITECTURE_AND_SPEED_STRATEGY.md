# 🚀 01. 공실뉴스 아키텍처 및 초고속 렌더링 전략 (GongsilNews Architecture & Speed Strategy)

본 문서는 사전에 논의된 기획(00_MASTER_PLAN)을 기술적으로 완벽하게 구현하기 위한 **엔지니어링 아키텍처와 비즈니스 로직 설계안**입니다.
이 프로젝트의 모든 코드는 **"사이트 및 데이터가 사용자에게 가장 빠르게(0초에 가깝게) 뜨는가?"** 라는 단 하나의 궁극적인 목표(KPI)를 지향합니다.

---

## 🎯 1. 핵심 지향성 및 목표 (Core Direction)
1. **극강의 체감 속도 보장**: Next.js의 서버 사이드 렌더링(SSR) 및 PPR을 활용하여 브라우저의 계산 부하를 없애고 빛의 속도로 화면을 제공한다.
2. **B2B SaaS 기반의 멀티 테넌트**: 하나의 소형 코드로 수백 개의 부동산별 서브도메인 홈페이지(`부동산명.gongsilnews.com`)를 독립적으로 운영한다.
3. **지도 API 비용 최소화 및 성능 극대화**: 프론트엔드 연산을 줄이고 데이터베이스 중심의 아키텍처(On-Write Geocoding, RLS)로 전환한다.

---

## 🛠️ 2. 아키텍처 및 렌더링 최적화 방법론

### A. 하이브리드 렌더링 & PPR (Partial Prerendering) 연동
* **구조 분리**: 변하지 않는 레이아웃(헤더, 푸터, 메뉴)과 무거운 데이터(뉴스 리스트, 공실 지도)를 완전히 분리한다.
* **PPR 적용**: 레이아웃은 클라이언트 접속 즉시(0초) 보여주고, 무거운 데이터는 서버에서 가져오는 대로 부드럽게 끼워 넣는 스트리밍(Streaming) 렌더링을 적용한다.

### B. 서브도메인 테마 관리를 위한 CSS Variables
* **문제점**: Tailwind CSS에서 동적으로 클래스(`bg-[#ff0000]`)를 주입하면 성능 저하나 스타일 누락(Purge)이 발생함.
* **해결책**: Next.js Middleware가 접속한 서브도메인(`happy`)을 감지하여 DB에서 해당 업소의 테마 컬러를 1회만 조회. 해당 컬러를 `<body>`에 CSS 변수(`--theme-primary`) 형태로 주입하여, Javascript 연산 없이 순수 CSS 처리만으로 테마를 즉시 전환.

### C. 지도 구동 및 Geocoding(좌표 변환) 백엔드 분리
* **비용 절감**: 지도를 열 때마다 주소를 좌표로 텍스트 변환하면 카카오/네이버 지도 API 요금 폭탄 및 지도 로딩 지연 발생.
* **저장 시점 변환 (On-Write)**: 사용자가 매물 등록(우편번호 검색)을 마치는 순간 백그라운드에서 단 1회 API를 호출해 위도(`lat`)와 경도(`lng`)를 얻음. 화면엔 보이지 않지만 DB 컬럼에 숫자로 영구 저장.
* **불러오기**: 손님이 공실 지도를 켤 때, 변환 과정 없이 DB에 있는 순수 위경도 좌표들만 통째로 던져주어 렌더링 체감 속도를 한계까지 끌어올림. 나아가 Supabase **PostGIS**를 활용해 수만 개의 매물 중 "현재 내 화면 반경 내" 매물만 즉시 추출.

## 🔐 3. B2B 인증 (Auth) 및 RLS 철벽 보안 구조

### A. 초간편 소셜 로그인 통합 연동
* 이탈률 방지를 위해 **구글, 카카오, 네이버 1초 간편 로그인(OAuth)** 체제로 일원화. (Supabase Auth 기반) 

### B. 가입 및 조직 승인 파이프라인
* **일반 회원(USER)**: 소셜 로그인 후 즉시 포털을 관람하고 찜하기 기능 사용.
* **부동산 직원(MEMBER)**:
  1. 서브도메인 도는 포털에서 로그인 후 '소속 부동산' 검색 및 가입 신청.
  2. 시스템상 상태는 `승인 대기(Pending)`. 글 작성 권한 없음.
  3. 해당 부동산의 소장(TENANT_ADMIN)이 관리자 페이지에서 로그인 알림을 받고 **직원을 [승인]**.
  4. 상태가 `승인됨(Approved)`으로 변경되며 관리 자격 확보.

### C. Supabase RLS (데이터 격리 방위선)
* 데이터베이스 테이블(RLS 규정) 레벨에서 다음을 원천 차단:
  * "이 매물(로우)의 agency_id 와, 지금 토큰을 들고 있는 유저 소속의 agency_id 가 일치하는가?"
  * "이 유저의 승인 단계가 완료되었는가?"
* 프론트엔드가 아닌 DB 본연에서 방어하므로 해킹 및 타사 데이터 무단 수정 100% 방지.

## 📰 4. 언론사형 시스템 (공실뉴스 기자단 워크플로우)

### A. 기사 송고 및 승인
* 최고관리자에게 사전에 **'기자 (에디터)' 권한**을 부여받은 회원(중개사, 일반)만이 글쓰기 가능.
* 기사 작성 완료 시 즉시 포털에 퍼블리싱 되지 않고 상태가 `대기중(Pending)` 으로 DB에 저장.
* 본사(최고관리자)의 데스킹(팩트체크 및 오타수정)을 거친 후 **최종 승인 버튼**을 눌러야만 `발행(Published)` 처리됨.

## 🧲 5. B2B2C 매물 위탁망 및 영업 킬러 모델 (Lead Generation)

이 시스템의 비즈니스적 핵심은 일반인(임대인)의 매물을 중개사들의 수익(영업망)으로 직결시켜주는 구조에 있습니다.

### A. 임대인 주도형 매물 타겟팅 (Consignment Routing)
* 일반인이 포털에서 "원룸 내놓기"를 할 때, 자신의 매물이 노출될 대상(부동산)을 능동적으로 선택할 수 있습니다.
  * **전체 노출망**: 가입된 모든 중개사 서브도메인에 노출.
  * **지역 기반(Geo-Fencing)**: 입력한 주소 반경 Xkm 이내의 동네 부동산 서브도메인에만 노출.
  * **지정 위탁**: 특정 부동산(예: 행복부동산 등)을 직접 검색하여 독점 형태 혹은 공동 형태로 노출.
* **DB 구조**: 매물과 부동산 간 다대다(N:M) 연결 테이블(`property_consignments`)을 통해 관리되며, 서브도메인별 접속 시 해당 테이블을 참조하여 권한이 있는 매물만 정확하게 렌더링.

### B. 연락처 실시간 마스킹 (Owner Info Masking)
* 일반인이 등록한 매물이라도, 손님이 특정 부동산 홈페이지(`happy.gongsilnews.com`)에 접속해서 해당 매물을 볼 때는 집주인의 실명 구글 연동 연락처가 곧바로 노출되지 않습니다.
* 화면에는 집주인 대신 **해당 서브도메인의 소유자인 중개사의 연락처 및 프로필로 뒤집어씌워(Override) 노출**시킵니다.
* **효과**: 가맹 부동산들은 직접 매물을 수집하지 않아도 본사 망(일반인 풀)에서 들어온 매물이 자동으로 자기 매물인 것처럼 표시되며 손님(Lead)을 확보할 수 있는 압도적인 혜택을 누립니다.

## 📱 6. 모바일 네이티브 UX 및 실시간 소통 (Mobile & Real-time)

### A. 네이티브 앱 급 체감 속도 (PWA 도입)
* **문제점**: 일반적인 웹사이트는 매번 클릭할 때마다 흰 화면이 뜨며 로딩해야 해서 앱 대비 체감이 느림.
* **해결책**: PWA(Progressive Web App) 기술의 Service Worker를 도입. 사용자가 최초 접속 시 사이트의 껍데기(헤더, 아이콘 등)를 휴대폰 스토리지에 캐싱.
* **효과**: 이후 클릭부터는 화면 전환 지연(흰 화면) 없이 즉각적으로 넘어가며, 앱 아이콘 설치 및 주소창 숨김을 통해 사용자는 네이티브 앱(AOS/iOS)을 쓰는 것과 100% 동일한 부드러운 속도를 경험함.

### B. 회원 간 실시간 쪽지 연동 (Supabase WebSockets)
* **방식**: 새로고침을 해야만 쪽지를 확인할 수 있는 구시대적 방식 탈피.
* **기술**: Supabase의 핵심 기능인 `Realtime (PostgreSQL 논리적 복제통신)`을 활용.
* **효과**: 손님이 "방 언제 볼 수 있나요?"를 보내는 순간, 새로고침 없이 중개사 화면에 카카오톡처럼 애니메이션과 함께 실시간(0.1초 핑 기준)으로 쪽지 팝업 아이콘 알림이 점등됨. 사이트 체류 시간 및 전환율 극대화.

## 🤖 7. AI 인공지능 기반 업무 자동화 (AI Auto-Filler & Reporter)

사용자(중개사, 기자, 일반인)가 콘텐츠를 등록하는 과정을 기존 대비 3배 이상 단축시키는 마법 같은 AI API 연동 전략입니다.

### A. 매물 등록 초간편 자동 채우기 (AI Auto-Filler)
* **자연어 메모 입력**: 중개사가 "강남역 풀옵션 원룸 보증금 1천 월 80 에어컨 있음"처럼 대충 문장으로 치거나 음성으로 던지면, AI(NLP)가 이를 분석해 보증금, 월세, 옵션 체크박스를 자동으로 모두 채워줍니다. (수작업 입력 노동 해방)
* **사진 시각 인식(Vision AI)**: 방 사진이나 전단지를 올리면 AI가 사진을 판독하여 "남향", "풀옵션", "깔끔한 인테리어" 등의 키워드를 추출 및 홍보 매물 코멘트를 3초 만에 대신 작성해 줍니다.

### B. 뉴스 기사 초안 자동 작성 (AI Reporter)
* **URL / 키워드 기반 작성**: 외부 부동산 보도자료나 부동산원 지표의 링크(URL)만 붙여넣어도, 저작권 문제가 없도록 AI가 공실뉴스만의 톤앤매너로 기사를 새롭게 작성하고 3줄 요약을 첨부합니다.
* **활용 효과**: 일반 회원과 중개사들은 글솜씨가 없어도 키워드 몇 개로 프로 기자처럼 기사를 송고하게 되며, 이는 플랫폼의 뉴스 콘텐츠(UGC)를 기하급수적으로 폭발시키는 원동력이 됩니다. 데스킹 시 관리자의 피로도 또한 크게 감소합니다.
