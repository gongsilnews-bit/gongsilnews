# 부동산회원 승인 파이프라인 (Realtor Approval Workflow) 완료 🚀

요청하신 "부동산회원 승인(대기/반려/정상) 및 알림 플로우" 연동을 모두 완료하였습니다! 
지시해주신 대로 **"정상 승인된 회원은 다시 저장 버튼을 눌러도 승인대기(PENDING) 상태로 강등되지 않도록"** 로직을 단단히 고정해 두었습니다.

## 🛠 주요 작업 내용

### 1. `최고 관리자` 패널 연동 (`/admin`)
- 회원목록(Table)의 `가입 완료 여부` 컬럼이 단순히 "정상/승인대기" 텍스트만 보여주던 것에서, 실제 중개업소(`agencies.status`) 데이터를 가져와서 실시간 매핑하도록 기능을 업그레이드 하였습니다. (정상승인/승인대기/서류보완 필요)
- **강제 상태 변경:** 관리자 모드로 `[회원수정]` 폼을 열람 시, `승인대기(PENDING)`, `정상승인(APPROVED)`, `서류보완(REJECTED)` 등 원하는 상태로 강제 전환할 수 있는 `<select>` 드롭다운을 활성화했습니다.

### 2. `부동산 회원` 대시보드 알림 (`/realty_admin`)
- 로그인 즉시 본인의 `현재 승인 상태`를 DB에서 체크하도록 연동했습니다.
- **배너 노출 조건:**
    - `승인대기 (PENDING)` 상태인 경우 👉 🟡 메인 대시보드 상단에 **[현재 서류 검토가 진행 중입니다]** 안내 경고 노출
    - `서류보완 (REJECTED)` 상태인 경우 👉 🔴 메인 대시보드 상단에 **[제출하신 서류의 보완이 필요합니다]** 긴급 안내 노출 및 정보설정 탭 가이드
    - `정상승인 (APPROVED)` 상태인 경우 👉 배너와 긴급뱃지가 모두 사라지며 완전한 정상 UI 노출

### 3. `정보설정(폼)` 상태 보존 로직 (`MemberRegisterForm.tsx`)
- 일반 사용자가 `[정보설정]` 화면에 접속 시, 승인 상태는 수정할 수 없는 **읽기 전용 뱃지**로만 나타납니다.
- `저장하기`를 눌렀을 때의 덮어쓰기 로직:
  - **신규 가입 유저:** 무조건 `PENDING`(승인대기)
  - **서류보완 상태 유저:** 서류를 재첨부하고 누르면 다시 검토를 받아야 하니 `PENDING`(승인대기)으로 자동 전환
  - **💡 정상승인 유저:** 지시사항에 따라, 단순 텍스트 변경 후 저장을 눌러도 상태를 `APPROVED`(정상)로 유지하여 재승인 스트레스를 제거했습니다!

## 👀 체크 후 다음 플로우
이제 관리자가 "서류보완"이나 "정상승인"을 주었을 때, 시스템이 각 회원에게 이메일이나 카카오 알림톡 등을 날려주는 기능까지 붙이시면 파이프라인이 더욱 완벽해질 것 같습니다. 확인해 보시고, 추가로 보완/수정할 디자인이나 기능이 있다면 말씀해 주세요!
