# 💡 IDEA 10: 관리자 페이지(Admin) 리팩토링 및 성능 최적화 전략

본 문서는 운영 효율성과 렌더링 성능을 극대화하기 위해, 거대해진 관리자 페이지(Admin)의 구조를 Next.js 환경에 맞게 레고형(컴포넌트 분리)으로 리팩토링하는 핵심 전략을 기록한 문서입니다.

---

## 1. 관리자 페이지 전면 컴포넌트 분리 (레고형 아키텍처 도입)

현재 관리자 페이지 단일 파일(`src/app/admin/page.tsx`)이 방대한 코드를 담고 있어, 잦은 리렌더링(화면 버벅임)과 유지보수 지옥이 예상됩니다. 이를 Next.js에 맞게 분리해야 합니다.

### 🧱 3단계 조립식 분리 전략

**1단계: 레이아웃 분리 (껍데기 분리)**
- 좌측 사이드바와 상단 헤더, 다크모드 로직 등은 `src/app/admin/layout.tsx`로 빼서 공통 베이스로 만듭니다.
- 내부 알맹이(콘텐츠)가 바뀔 때 무거운 사이드바와 헤더가 다시 렌더링되는 것을 원천 차단합니다.

**2단계: 아이콘 및 UI 공통 로직 분리 (다이어트)**
- 파일 상단에 수백 줄을 차지하는 SVG 아이콘 태그들은 `components/admin/icons/AdminIcons.tsx` 등 독립된 파일로 축출하여 가볍게 만듭니다.

**3단계: 메뉴별 라우팅 독립 분리 (가장 핵심)**
- 현재 `activeMenu === "dashboard" ? A : B` 형태로 단일 파일에서 컴포넌트를 토글시키는 방식을 버립니다.
- Next.js의 폴더 기반 라우팅을 적극 활용하여 화면별로 진입점을 완전히 쪼갭니다.
  - 대시보드: `admin/dashboard/page.tsx`
  - 공실관리: `admin/gongsil/page.tsx`
  - 기사관리: `admin/article/page.tsx`
  - 스터디관리: `admin/study/page.tsx`
  - 회원관리: `admin/members/page.tsx`
- 이렇게 분할하면 로딩 속도(Code Splitting)가 비약적으로 빨라지고, 특정 관리자 페이지에서 생긴 오류가 서비스 전체를 마비시키지 않습니다.
